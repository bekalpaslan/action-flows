# Phase 4 -- UI Design Contract

> Visual and interaction contract for the layout and navigation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase replaces the fixed CSS Grid layout (AppShell.css: `grid-template-columns: 220px 1fr 300px`) with a resizable 3-panel layout using react-resizable-panels. It rebuilds the sidebar with collapse/expand behavior, splits the workspace into pipeline and content regions, and adds a command palette. All values inherit from the Phase 3 token system (theme.css).

## Spacing Scale

Inherited from Phase 3 design contract. The full 4px-base scale is defined in theme.css `--spacing: 4px`. Phase 4 uses the following spacing values for layout-specific concerns:

| Token | Value | Tailwind Class | Phase 4 Usage |
|-------|-------|----------------|---------------|
| 0 | 0px | p-0 | Collapsed panel gaps |
| 1 | 4px | p-1 | Drag handle hit area padding |
| 2 | 8px | p-2, gap-2 | Sidebar item vertical gap, icon-to-label gap |
| 3 | 12px | p-3 | Sidebar padding (collapsed state) |
| 4 | 16px | p-4, gap-4 | Sidebar padding (expanded state), workspace content padding |
| 6 | 24px | p-6, gap-6 | Section padding inside workspace content area |
| 8 | 32px | p-8 | Page-level layout margins |
| 11 | 44px | h-11, w-11 | Sidebar item touch target height, command palette item height |

Exceptions:
- Drag handle visible area: 4px wide (1px visible bar + 3px transparent hit extension on each side = 10px total hit zone). Not a 4px-multiple hit zone because the visual element is the constraint -- the hit extension is transparent padding.
- Sidebar collapsed width: 64px (w-16) -- fits icon (20px) + padding (12px each side) + accent border (4px)
- Sidebar expanded width: default 240px (--sidebar-expanded), min 200px, max 320px

**Source:** Phase 3 UI-SPEC spacing scale, LAYOUT-01/LAYOUT-04 constraints

## Interaction Contracts

### Panel Resize

| Interaction | Behavior |
|-------------|----------|
| Drag handle hover | Handle bar becomes visible (border-strong at 50% opacity) |
| Drag handle active | Handle bar turns accent at 30% opacity, cursor locks to col-resize/row-resize |
| Drag to min | Panel snaps to collapsed state if collapsible, otherwise stops at min% |
| Drag from collapsed | Panel expands from minimum viable size (not from 0% -- starts at min%) |
| Double-click handle | Toggle adjacent collapsible panel between collapsed and default size |
| Keyboard | Tab to handle, Arrow keys to resize in 1% increments, Home/End for min/max |

### Sidebar Collapse/Expand

| Interaction | Behavior |
|-------------|----------|
| Drag sidebar handle below 8% | Sidebar snaps to collapsed (icon-only, ~64px) |
| Click collapse button (expanded) | Sidebar animates to collapsed state |
| Click expand button (collapsed) | Sidebar animates to expanded state (restores previous %) |
| Click workbench item (collapsed) | Switches workbench AND expands sidebar |
| Keyboard shortcut 1-7 | Switches to workbench by number (works globally when palette is closed) |
| Keyboard Ctrl+B | Toggle sidebar collapse/expand |

### Chat Panel Collapse/Expand

| Interaction | Behavior |
|-------------|----------|
| Drag chat handle below 10% | Chat panel collapses fully (0px), workspace takes remaining space |
| Drag from collapsed (click handle area) | Chat expands to previous size or default 25% |
| Keyboard Ctrl+Shift+C | Toggle chat panel collapse/expand |

### Command Palette

| Interaction | Behavior |
|-------------|----------|
| Cmd/Ctrl+K | Open palette, focus input, show all commands |
| Type in input | Filter commands by fuzzy match on label |
| Select navigation item | Switch workbench, close palette |
| Select toggle action | Execute toggle (sidebar/chat/pipeline), close palette |
| Click outside / Escape | Close palette, return focus to previously focused element |
| Open when already open | Close palette (toggle behavior) |

## Accessibility Contract

Inherits Phase 3 global accessibility rules. Phase 4 adds:

| Requirement | Implementation |
|-------------|----------------|
| Panel resize keyboard | Tab to drag handle, Arrow keys resize, Home/End for min/max (built into react-resizable-panels) |
| Drag handle ARIA | `role="separator"` with `aria-orientation` and `aria-valuenow` (react-resizable-panels provides this) |
| Sidebar navigation | `<nav aria-label="Workbench navigation">` with `role="list"` items |
| Active workbench | `aria-current="page"` on active sidebar item |
| Collapsed sidebar tooltips | Tooltip content matches workbench label for screen reader parity |
| Command palette | `role="dialog"` with `aria-label="Command palette"`. cmdk provides `role="listbox"` for items. |
| Command palette input | `aria-label="Search commands"`, `aria-expanded="true"`, `aria-controls` pointing to results listbox |
| Keyboard shortcuts | Not trapped -- Escape always closes overlay, Tab cycles through focusable elements within open palette |
| Skip link | Consider adding `role="main"` to workspace content area for skip-nav compatibility |
| Reduced motion | Panel animations respect `prefers-reduced-motion` -- transitions set to 0ms |
| Focus return | When command palette closes, focus returns to the element that triggered it |
| Sidebar collapse | Collapse/expand state announced via `aria-expanded` on toggle button |

# Phase 999.1 Plan 02: LedgerService & summarizeTrace Summary

**One-liner:** Append-only JSONL LedgerService (append/appendBatch/query) and pure summarizeTrace(GateTrace)->LedgerEntry mapper, turning Wave 0 ledger and gateTraceSummarizer tests GREEN.

## What Was Built

### Task 1 — LedgerService (commit 44344e3)

`packages/backend/src/services/ledgerService.ts` — 67 lines.

- `new LedgerService(filePath?)` — optional path; defaults to `path.join(process.cwd(), 'data', 'ledger.jsonl')`.
- `append(entry)` — single JSON line append, lazy `mkdirSync` on parent dir.
- `appendBatch(entries)` — multi-line append in one `appendFileSync` call; no-op on empty array.
- `query(filter)` — reads full file, parses each line (skipping malformed), filters by gateId, outcome, fromTimestamp, toTimestamp, applies `slice(-limit)`.
- Module-level `ledgerService` singleton exported for Plan 03/05 wiring.
- Log prefix `[LedgerService]` on batch append success and on warn paths.

Implements D-03 (fields) and D-04 (path).

### Task 2 — summarizeTrace (commit 8f30c74)

`packages/backend/src/services/gateTraceSummarizer.ts` — 29 lines.

- Pure function, no side effects, no runtime service imports.
- Returns `null` for traces missing `gateId`, `chainId`, or `timestamp`.
- Maps `validationResult?.passed === true` -> `'pass'`, `=== false` -> `'fail'`, otherwise `'unknown'`.
- `reason = (trace.rationale ?? trace.selected ?? '').slice(0, 200)`.
- `timestamp` set to promotion time (`new Date().toISOString()`); `sourceTraceTimestamp` preserves original trace timestamp for provenance.

Implements D-06.

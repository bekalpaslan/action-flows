# Phase 999.1 Plan 04: Learnings Archive Lifecycle Summary

**One-liner:** CRLF-safe LearningsArchiver caps active LEARNINGS.md at 50 entries and append-archives overflow to year-scoped LEARNINGS-archive-YYYY.md files on every CleanupService daily run.

## What Was Built

### Task 1 — LearningsArchiver service (commit `003e580`)

- New `packages/backend/src/services/learningsArchiver.ts` (~95 lines).
- `new LearningsArchiver(activePath?)` defaults to `<cwd>/../../.claude/actionflows/LEARNINGS.md`.
- `archiveIfNeeded()` is a no-op when entry count <= 50.
- Split algorithm: `raw.split(/(?=^### L\d+:)/m)` with preamble + entry chunks; CRLF normalized first (Pitfall 2, MEMORY.md L009).
- When overflow: takes oldest `count - 50` entries, groups by year parsed from `**Date:** YYYY-MM-DD`, appends each year group to `LEARNINGS-archive-YYYY.md` (creating with `# Learnings Archive YYYY` header if new), then rewrites active file with preamble + newest 50 entries using LF endings.
- `.gitattributes` extended with `.claude/actionflows/LEARNINGS.md text eol=lf` and archive glob, defending the CRLF regex trap globally.
- Exports `learningsArchiver` singleton.
- Both test cases in `packages/backend/src/__tests__/learningsArchiver.test.ts` are GREEN.

### Task 2 — CleanupService wiring (commit `512fda9`)

- `cleanup.ts` gains:
  - `import type { LearningsArchiver }`
  - private `learningsArchiver: LearningsArchiver | null` field
  - `setLearningsArchiver(archiver)` setter
  - `private async archiveLearnings()` with try/catch + "not wired" skip log
  - `runCleanup()` now calls `await this.archiveLearnings()` after `await this.pruneGateTraces()` and before `getStats`
- `index.ts`:
  - New `import { learningsArchiver } from './services/learningsArchiver.js'`
  - `cleanupService.setLearningsArchiver(learningsArchiver)` added after existing `setPruneDeps` and before `cleanupService.start()`
- `pnpm --filter @afw/backend type-check` passes.

## Runtime Order in `runCleanup()`

1. `filePersistence.cleanupOldFiles()`
2. `await this.pruneGateTraces()` (from Plan 03)
3. `await this.archiveLearnings()` (this plan)
4. `filePersistence.getStats()` log

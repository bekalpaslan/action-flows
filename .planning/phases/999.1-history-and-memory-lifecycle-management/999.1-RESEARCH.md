# Phase 999.1: History & Memory Lifecycle Management - Research

**Researched:** 2026-04-10
**Domain:** Backend lifecycle management, JSONL append-only stores, file-based retention, ledger patterns
**Confidence:** HIGH

## Summary

This phase adds a 3-layer history model (raw → ledger → canonical) to three unbounded stores: `gate-traces.jsonl` (529 lines, growing), `.claude/actionflows/logs/` (14MB, inconsistent INDEX.md), and `LEARNINGS.md` (261 lines, no cap). The good news: every extension point already exists. `CleanupService` has a running daily scheduler hook; `HealthScoreCalculator` already owns the gate-traces JSONL file and already does a 7-day cutoff on load; `LearningsBrowser` (Phase 10) already surfaces approved healing attempts as provisional learnings. The work is wiring, not invention.

**Primary recommendation:** Extend existing services in place. Create one new service (`LedgerService`) that owns `ledger.jsonl` writes and reads, extend `CleanupService.runCleanup()` to orchestrate gate-trace pruning + LEARNINGS cap enforcement, add `ledger` sub-routes to the existing `routes/history.ts`, and add a log-dir-creation hook to the orchestrator's `create-log-folder` path. Do not build generic log-rotation libraries — the existing JSONL+load-with-cutoff pattern is exactly right.

### Pattern 1: Append-Only JSONL (established)

**What:** One JSON object per line, `fs.appendFileSync(path, JSON.stringify(obj) + '\n')`. Load-time filter applies TTL cutoff.

**Source:** `packages/backend/src/services/healthScoreCalculator.ts` lines 54-90 (`loadPersistedTraces`, `persistTrace`). This is the exact pattern — reuse it verbatim for `ledger.jsonl`.

```typescript
// From healthScoreCalculator.ts:82-90 — copy this pattern
private persistTrace(trace: GateTrace): void {
  try {
    const dir = path.dirname(this.traceFilePath);
    if (!fs.existsSync(dir)) fs.mkdirSync(dir, { recursive: true });
    fs.appendFileSync(this.traceFilePath, JSON.stringify(trace) + '\n');
  } catch (err) {
    console.warn('[HealthScore] Could not persist trace:', err);
  }
}
```

### Anti-Patterns to Avoid

- **Don't rewrite `gate-traces.jsonl` on prune.** Rewriting the whole file on every prune is O(n) I/O. Instead, stream-read, filter-write to a temp file, atomic-rename. Or accept the rewrite cost since it runs once per day.
- **Don't parse LEARNINGS.md with regex only.** Entries are `### LNNN:` headers followed by bullet fields. Split on `/^### L\d+:/m` multiline, keep header + body together. CRLF on Windows matters (L009 in MEMORY.md) — use `/\r?\n/` in splits.
- **Don't store ledger entries in Redis/MemoryStorage.** D-04 explicitly locates ledger on disk alongside gate-traces. Storage KV interface is OK for "last prune timestamp" metadata only.
- **Don't summarize traces inline on every write.** Promotion runs only at TTL expiry (D-06) — a batch job, not a per-write cost.

## Common Pitfalls

### Pitfall 1: Rewriting gate-traces.jsonl during prune loses in-flight writes

**What goes wrong:** `HealthScoreCalculator.persistTrace()` appends to the same file CleanupService is rewriting. A race condition can lose entries.

**Why it happens:** JSONL append is not atomic with respect to a concurrent rewrite.

**How to avoid:** Add a `prune()` method **on HealthScoreCalculator itself** (not on CleanupService). It owns the file and can coordinate with the in-memory buffer:
1. Snapshot `traceBuffer` state
2. Write filtered buffer atomically (`fs.writeFileSync` to temp → rename)
3. Any trace arriving during the rewrite is appended to the new file post-rename

**Warning signs:** Traces appearing in console logs but missing from the loaded buffer after a cleanup run.

### Pitfall 2: CRLF breaks LEARNINGS.md regex parsing on Windows

**What goes wrong:** `/^### L\d+:/m` under Windows `core.autocrlf=true` doesn't anchor correctly with CRLF endings. (MEMORY.md L009 documents this exact trap.)

**How to avoid:** Normalize to LF before splitting: `content.replace(/\r\n/g, '\n')`. Consider adding `LEARNINGS.md` to `.gitattributes` as `text eol=lf` like the contract files.

### Pitfall 3: `healthScoreCalculator` loads the trace file at construction time

**What goes wrong:** Pruning that happens *after* startup doesn't free memory from the loaded `traceBuffer`.

**How to avoid:** `prune()` must update both the file AND `this.traceBuffer`. Don't just delete file lines.

### Pitfall 4: LEARNINGS.md has no strict numbering contract

**What goes wrong:** Entries are `L001`, `L002`, ..., but the archive may break continuity. New entries should keep incrementing from the highest number across **all** files (active + archives).

**How to avoid:** When archiving, scan active file for the highest `LNNN` before the first entry to archive; when surfacing new learnings, scan active for highest and increment. Don't restart numbering in the archive.

### Pitfall 5: Loading 14MB of logs is not the goal

**What goes wrong:** Someone reads the phase as "index all 14MB of existing logs." It's not — D-11 explicitly says old dirs without INDEX.md are accepted as legacy. **No backfill.**

**How to avoid:** The INDEX.md enforcement hook runs only at *creation time* of new log session dirs. The hook lives in the orchestrator's `create-log-folder` action (outside `packages/backend/` — likely in `.claude/actionflows/actions/` or the action runner).

### Pitfall 6: Ledger entry with missing fields during promotion

**What goes wrong:** Some `GateTrace` entries lack `validationResult` or `stepId`. Summarizer throws or produces garbage.

**How to avoid:** Write the summarizer as a pure function with explicit defaults: `outcome: trace.validationResult?.passed ?? null`, `reason: trace.rationale ?? ''`. Skip entries that lack required identity fields (`gateId`, `chainId`, `timestamp`) and log them under `[GateTracePruner] Skipped N malformed traces`.

### Example 2: Pure summarizer function

```typescript
// packages/backend/src/services/gateTraceSummarizer.ts — NEW FILE
import type { GateTrace, LedgerEntry } from '@afw/shared';

export function summarizeTrace(trace: GateTrace): LedgerEntry | null {
  if (!trace.gateId || !trace.chainId || !trace.timestamp) return null;

  const passed = trace.validationResult?.passed;
  const outcome: LedgerEntry['outcome'] =
    passed === true ? 'pass' : passed === false ? 'fail' : 'unknown';

  const reason = (trace.rationale ?? trace.selected ?? '').slice(0, 200);

  return {
    timestamp: new Date().toISOString() as LedgerEntry['timestamp'],
    chainId: trace.chainId,
    gateId: trace.gateId,
    gateName: trace.gateName,
    outcome,
    reason,
    sourceTraceTimestamp: trace.timestamp,
  };
}
```

### Example 3: LedgerService sketch

```typescript
// packages/backend/src/services/ledgerService.ts — NEW FILE
import fs from 'fs';
import path from 'path';
import type { LedgerEntry, LedgerFilter } from '@afw/shared';

export class LedgerService {
  private readonly filePath: string;

  constructor() {
    this.filePath = path.join(process.cwd(), 'data', 'ledger.jsonl');
  }

  append(entry: LedgerEntry): void {
    try {
      const dir = path.dirname(this.filePath);
      if (!fs.existsSync(dir)) fs.mkdirSync(dir, { recursive: true });
      fs.appendFileSync(this.filePath, JSON.stringify(entry) + '\n');
    } catch (err) {
      console.warn('[LedgerService] Append failed:', err);
    }
  }

  appendBatch(entries: LedgerEntry[]): void {
    if (entries.length === 0) return;
    const payload = entries.map(e => JSON.stringify(e)).join('\n') + '\n';
    try {
      const dir = path.dirname(this.filePath);
      if (!fs.existsSync(dir)) fs.mkdirSync(dir, { recursive: true });
      fs.appendFileSync(this.filePath, payload);
      console.log(`[LedgerService] Appended ${entries.length} entries`);
    } catch (err) {
      console.warn('[LedgerService] Batch append failed:', err);
    }
  }

  query(filter: LedgerFilter = {}): LedgerEntry[] {
    if (!fs.existsSync(this.filePath)) return [];
    const lines = fs.readFileSync(this.filePath, 'utf-8').trim().split('\n').filter(Boolean);
    let entries: LedgerEntry[] = [];
    for (const line of lines) {
      try { entries.push(JSON.parse(line)); } catch { /* skip */ }
    }
    if (filter.gateId) entries = entries.filter(e => e.gateId === filter.gateId);
    if (filter.outcome) entries = entries.filter(e => e.outcome === filter.outcome);
    if (filter.fromTimestamp) entries = entries.filter(e => e.timestamp >= filter.fromTimestamp!);
    if (filter.toTimestamp) entries = entries.filter(e => e.timestamp <= filter.toTimestamp!);
    if (filter.limit) entries = entries.slice(-filter.limit);
    return entries;
  }
}
```

### Example 4: HealthScoreCalculator.prune() extension

```typescript
// packages/backend/src/services/healthScoreCalculator.ts — ADD METHOD
/**
 * Prune traces older than 7 days. Returns the pruned traces so the caller
 * (CleanupService) can promote them to the ledger before they're dropped.
 */
public prune(): GateTrace[] {
  const cutoff = Date.now() - this.WINDOW_7D_MS;
  const survivors: GateTrace[] = [];
  const pruned: GateTrace[] = [];
  for (const t of this.traceBuffer) {
    if (new Date(t.timestamp).getTime() >= cutoff) survivors.push(t);
    else pruned.push(t);
  }
  this.traceBuffer = survivors;

  // Atomic rewrite of the file
  try {
    const tmpPath = this.traceFilePath + '.tmp';
    const content = survivors.map(t => JSON.stringify(t)).join('\n') + (survivors.length ? '\n' : '');
    fs.writeFileSync(tmpPath, content);
    fs.renameSync(tmpPath, this.traceFilePath);
  } catch (err) {
    console.warn('[HealthScore] Prune file rewrite failed:', err);
  }
  console.log(`[HealthScore] Pruned ${pruned.length} trace(s), ${survivors.length} retained`);
  return pruned;
}
```

### Example 5: LEARNINGS archive split (sketch)

```typescript
// packages/backend/src/services/learningsArchiver.ts — NEW FILE
import fs from 'fs';
import path from 'path';

const LEARNINGS_PATH = path.join(process.cwd(), '..', '..', '.claude', 'actionflows', 'LEARNINGS.md');
const MAX_ACTIVE_ENTRIES = 50;

interface Entry { header: string; body: string; dateYear?: number; }

export class LearningsArchiver {
  archiveIfNeeded(): void {
    if (!fs.existsSync(LEARNINGS_PATH)) return;
    const raw = fs.readFileSync(LEARNINGS_PATH, 'utf-8').replace(/\r\n/g, '\n');
    const parts = raw.split(/(?=^### L\d+:)/m);
    const preamble = parts[0] ?? '';
    const entries: Entry[] = parts.slice(1).map(chunk => {
      const header = chunk.split('\n')[0] ?? '';
      const body = chunk;
      const dateMatch = chunk.match(/\*\*Date:\*\*\s*(\d{4})-\d{2}-\d{2}/);
      return { header, body, dateYear: dateMatch ? parseInt(dateMatch[1]!, 10) : undefined };
    });

    if (entries.length <= MAX_ACTIVE_ENTRIES) return;

    const toArchive = entries.slice(0, entries.length - MAX_ACTIVE_ENTRIES);
    const toKeep = entries.slice(entries.length - MAX_ACTIVE_ENTRIES);

    // Group by year, append to year archives (oldest-first)
    const byYear = new Map<number, Entry[]>();
    for (const e of toArchive) {
      const y = e.dateYear ?? new Date().getFullYear();
      if (!byYear.has(y)) byYear.set(y, []);
      byYear.get(y)!.push(e);
    }
    for (const [year, items] of byYear) {
      const archivePath = path.join(path.dirname(LEARNINGS_PATH), `LEARNINGS-archive-${year}.md`);
      const header = fs.existsSync(archivePath) ? '' : `# Learnings Archive ${year}\n\n`;
      fs.appendFileSync(archivePath, header + items.map(i => i.body).join(''));
    }

    fs.writeFileSync(LEARNINGS_PATH, preamble + toKeep.map(i => i.body).join(''));
    console.log(`[LearningsArchiver] Archived ${toArchive.length} entries, ${toKeep.length} active`);
  }
}
```

3. **Ledger entry numbering / IDs?**
   - What we know: Gate traces use composite `harmony:gate:{chainId}:{gateId}:{timestamp}`. Ledger D-03 lists fields but no ID.
   - What's unclear: Do we need a ledger entry ID for the future search UI?

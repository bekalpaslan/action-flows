# Phase 999.1 Plan 03: Gate-trace TTL + ledger promotion (Summary)

Wired the 7-day TTL pruning loop for gate traces end-to-end: HealthScoreCalculator now owns a public `prune()` that atomically rewrites `data/gate-traces.jsonl`, and CleanupService promotes pruned traces to the ledger via `summarizeTrace` before they disappear. Dependencies are injected once at backend startup.

## Behavior

1. **TTL window:** 7 days (`WINDOW_7D_MS` in HealthScoreCalculator, matches `RETENTION_DAYS = 7` in file-persistence.ts).
2. **In-memory buffer:** partitioned into survivors vs pruned; `this.traceBuffer = survivors` before disk write.
3. **On-disk JSONL:** rewritten atomically (`<path>.tmp` → `fs.renameSync`).
4. **Ledger promotion:** pruned traces run through `summarizeTrace`; nulls (malformed) are counted and skipped; remainder appended in a single batch.
5. **DI guard:** if `pruneDeps` is null (tests / early startup), `pruneGateTraces()` logs a skip and returns silently.

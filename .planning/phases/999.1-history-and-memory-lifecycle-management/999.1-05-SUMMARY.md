# Phase 999.1 Plan 05: Ledger Query API Summary

One-liner: Added `GET /api/history/ledger` endpoint proxying to `LedgerService.query(filter)` with gateId/outcome/timestamp/limit filters, closing D-05.

## What Changed

Extended `packages/backend/src/routes/history.ts` with a single new route handler `router.get('/ledger', ...)` that:
- Reads query params `gateId`, `outcome`, `fromTimestamp`, `toTimestamp`, `limit`
- Coerces `limit` via `parseInt(..., 10)` and ignores NaN / non-positive values
- Validates `outcome` is `'pass' | 'fail'`, ignores otherwise
- Delegates to `ledgerService.query(filter)` singleton
- Returns `200 { entries: LedgerEntry[] }` or `500 { error }` on failure
- Imports `ledgerService` from `'../services/ledgerService.js'` and `LedgerFilter` type from `'@afw/shared'`

Route positioned before `export default router`, matches module-level `router` style of the file (no factory pattern).

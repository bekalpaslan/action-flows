Hook subscribes to WebSocket validation:violation events and shows toasts:

```typescript
import { useEffect } from 'react';
import { toast } from 'sonner';
import { wsClient } from '@/lib/ws-client';
import { useValidationStore } from '@/stores/validationStore';
```

Accept `workbenchId: WorkbenchId` parameter.

In useEffect (deps: [workbenchId]):
- Subscribe to workbenchId channel via `wsClient.subscribe(workbenchId, handler)`
- Handler checks `envelope.type === 'validation:violation'`
- On violation: cast payload to ViolationSignal, call `useValidationStore.getState().addViolation(workbenchId, violation)`

- Duration per severity: critical = `Infinity` (user must dismiss), warning = 8000, info = 5000
- Cleanup: unsubscribe on unmount

Also subscribe to `_system` channel for system-wide violation broadcasts (fallback channel from backend).

Export: `export function useViolationSignals(workbenchId: WorkbenchId): void`.

```typescript
const handleRevert = async (commitHash: string) => {
  try {
    const res = await fetch('/api/checkpoints/revert', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ commitHash }),
    });
    if (res.ok) {
      toast.success('Checkpoint reverted successfully');
    } else {
      const data = await res.json();
      toast.error(data.message || 'Revert failed');
    }
  } catch {
    toast.error('Revert failed -- the git operation could not complete. Check for uncommitted changes and try again.');
  }
};
```

**3. Create `packages/backend/src/services/checkpointService.ts`**:

Singleton class `CheckpointService` with methods:
- `listCheckpoints(limit?: number): Promise<CheckpointData[]>` -- runs `git log --oneline --format="%h|%s|%aI|%N" -n ${limit || 20}` via `child_process.execSync` (cwd = project root). Parse each line into `CheckpointData`. For `filesChanged`, run `git diff-tree --no-commit-id --name-only -r <hash>` and count lines. Return newest first.
- `revertToCheckpoint(commitHash: string): Promise<{ success: boolean; message: string }>` -- first validate hash exists with `git cat-file -t <hash>`. Check if it's a merge commit by counting parents (`git cat-file -p <hash>` and grep for `^parent`). If merge, use `git revert -m 1 --no-edit <hash>`. If regular commit, use `git revert --no-edit <hash>`. Return success/failure message. Per D-08: creates a NEW commit (not reset --hard).
- `getCheckpointForCommit(hash: string): Promise<CheckpointData | null>` -- get data for a single commit.

All git commands use `execSync` with `{ encoding: 'utf-8', cwd: process.cwd() }`. Wrap in try-catch, return meaningful errors. Log with `[CheckpointService]` prefix.

Export singleton: `export const checkpointService = new CheckpointService();`

**4. Create `packages/backend/src/services/approvalService.ts`**:

Singleton class `ApprovalService` with in-memory storage (Map):
- `autonomyLevels: Map<string, AutonomyLevel>` -- initialized with `DEFAULT_AUTONOMY_LEVELS` from shared
- `pendingRequests: Map<string, ApprovalRequest>` -- active approval requests
- `requestTimeouts: Map<string, NodeJS.Timeout>` -- auto-deny timers

Methods:
- `getAutonomyLevel(workbenchId: string): AutonomyLevel` -- returns level from map, defaults to 'supervised'
- `setAutonomyLevel(workbenchId: string, level: AutonomyLevel): void` -- updates the map
- `createRequest(params: { action: string; description: string; files?: string[]; workbenchId: string; sessionId: string }): ApprovalRequest` -- creates with `id = 'approval_' + Date.now()`, status 'pending', expiresAt = now + 120s. Starts 120s timeout that auto-resolves as 'timed_out'. Returns the request.
- `getRequest(id: string): ApprovalRequest | null`
- `resolveRequest(id: string, status: 'approved' | 'denied'): ApprovalRequest | null` -- sets status, resolvedAt, clears timeout. Returns null if not found or already resolved.
- `needsApproval(workbenchId: string, action: string): boolean` -- checks autonomy level. 'full' = never needs approval. 'restricted' = always needs. 'supervised' = only for destructive actions. Destructive actions per D-11: `['delete_files', 'remove_directory', 'force_push', 'drop_table', 'git_reset_hard']`.
- `cleanup()` -- clears all timeouts (for graceful shutdown)

Export singleton: `export const approvalService = new ApprovalService();`

- Broadcast each violation via WebSocketHub to the workbench channel: `hub.broadcastToChannel(workbenchId, JSON.stringify({ channel: workbenchId, type: 'validation:violation', payload: violationSignal, ts: new Date().toISOString() }))`.
- Access the hub instance from the request -- the hub is attached to the Express app or imported. Check how existing routes access the hub (look at index.ts for how hub is made available). If hub is not on `req.app`, import the `broadcastToChannel` utility from `../ws/handler.js` or pass hub via middleware.

**2. Create `packages/backend/src/routes/checkpoints.ts`**:

Express Router with:

`GET /` -- list recent checkpoints:
- Query param `limit` (default 20)
- Call `checkpointService.listCheckpoints(limit)`
- Return 200 `{ checkpoints: CheckpointData[] }`

`POST /revert` -- revert to a checkpoint:
- Body: `{ commitHash: string }`
- Validate commitHash is a non-empty string matching `/^[0-9a-f]{7,40}$/`
- Call `checkpointService.revertToCheckpoint(commitHash)`
- If success, broadcast via WebSocket to `_system` channel: type `checkpoint:reverted`, payload `{ commitHash }`
- Return 200 or 500 based on result

`GET /:hash` -- get single checkpoint data:
- Call `checkpointService.getCheckpointForCommit(hash)`
- Return 200 or 404

**3. Create `packages/backend/src/routes/approvals.ts`**:

Express Router with:

`GET /autonomy/:workbenchId` -- get autonomy level for workbench:
- Call `approvalService.getAutonomyLevel(workbenchId)`
- Return 200 `{ workbenchId, autonomyLevel }`

`PUT /autonomy/:workbenchId` -- set autonomy level:
- Body: `{ level: AutonomyLevel }`
- Validate level is one of 'full', 'supervised', 'restricted'
- Call `approvalService.setAutonomyLevel(workbenchId, level)`
- Return 200 `{ workbenchId, autonomyLevel: level }`

`POST /request` -- create approval request:
- Body: `{ action, description, files?, workbenchId, sessionId }`
- Check `approvalService.needsApproval(workbenchId, action)`
- If no approval needed, return 200 `{ approved: true, autoApproved: true }`
- If approval needed, call `approvalService.createRequest(body)`
- Broadcast `approval:request` via WebSocket to the workbench channel
- Return 201 `{ request: ApprovalRequest }`

`GET /:id/status` -- poll approval status (hooks use this):
- Call `approvalService.getRequest(id)`
- Return 200 `{ status, resolvedAt? }` or 404

`POST /:id/resolve` -- resolve an approval (frontend sends this):
- Body: `{ status: 'approved' | 'denied' }`
- Call `approvalService.resolveRequest(id, status)`
- Broadcast `approval:resolved` via WebSocket to the workbench channel
- Return 200 or 404

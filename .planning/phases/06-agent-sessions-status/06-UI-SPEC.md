# Phase 6 -- UI Design Contract

> Visual and interaction contract for the agent sessions and status phase. Generated by gsd-ui-researcher.
>
> This phase builds the frontend status panel and session lifecycle UI. The backend SessionManager is the sole owner of Agent SDK calls (CONTEXT.md D-03); the frontend communicates via WebSocket only. The primary frontend deliverable is a collapsible status panel (CONTEXT.md D-04) at the bottom of the workspace area, styled as a task manager, with manual start/stop controls (D-06) and toast notifications for session events (D-07). All visual values inherit from the Phase 3 design system (theme.css tokens, component library). This contract defines the phase-specific visual elements: status panel layout, session state indicators, toast patterns, animations, and interaction behaviors.

## Spacing Scale

Inherited from Phase 3 (03-UI-SPEC.md). One documented exception below.

Status panel-specific spacing applications:

| Context | Token | Value | Rationale |
|---------|-------|-------|-----------|
| Status panel internal padding | p-3 | 12px | 12px exception justified: compact panel context requires tighter spacing than standard 16px to preserve vertical real estate in a secondary status area |
| Table row height | h-10 | 40px | Comfortable touch target while keeping panel compact |
| Table cell horizontal padding | px-3 | 12px | 12px exception justified: compact panel context requires tighter column spacing without wasting horizontal space in a dense table layout |
| Table header/body gap | gap-0 | 0px | Header and body share continuous border grid |
| Action button gap | gap-2 | 8px | Tight gap between start/stop icon buttons in action column |
| Panel collapse handle height | h-8 | 32px | Minimal drag handle matching panel separator pattern |
| Toast internal padding | p-4 | 16px | Standard component padding per sonner defaults |
| Toast icon-text gap | gap-3 | 12px | 12px exception justified: compact panel context -- clear separation between status icon and message text in a constrained toast width |
| Status dot size | w-2 h-2 | 8px | Visible but unobtrusive inline status indicator |

Exceptions: 12px (p-3, px-3, gap-3) is used in this phase as a deliberate exception to the standard 8-point scale. Justification: the status panel is a compact, secondary UI region where 16px creates excessive whitespace and 8px feels too cramped for readable table content. 12px provides the right density for a task-manager-style panel.

## Color

Inherited from Phase 3 (03-UI-SPEC.md). Phase-specific color applications:

**Color distribution:** ~60% surface (panel background, table body), ~30% surface-2 (panel header, table header, row hover), ~10% accent/semantic (status dots, running icons, badges, start button).

**Primary focal point:** Status dot column and Loader2 spinner on running rows. These are the highest-contrast elements in the panel, drawing the eye to active session state. The accent-colored dot and spinning icon create the visual anchor for scanning session health at a glance.

All table rows use surface background to maintain the 60/30/10 ratio. Status is communicated through the colored dot, icon, and badge -- not through row background fills. Only the hovered row receives a subtle surface-2 background.

## Status Panel Design Contract

### Panel Layout (CONTEXT.md D-04: "task manager style")

| Property | Value |
|----------|-------|
| Position | Bottom of workspace content area, within the existing WorkspaceArea vertical PanelGroup |
| Integration | New Panel in WorkspaceArea's vertical Group: pipeline (top), content (middle), status panel (bottom) |
| Default size | 25% of workspace when expanded |
| Min size | 15% when expanded |
| Max size | 40% |
| Collapsed size | 0% (fully collapsed, only separator visible) |
| Collapsible | Yes -- collapses downward |
| Default state | Collapsed (opens on first session activity or manual toggle) |
| Background | var(--color-surface) |
| Border | 1px solid var(--color-border-strong) on top edge |

### Panel Header (always visible when expanded)

| Property | Value |
|----------|-------|
| Height | 32px |
| Background | var(--color-surface-2) |
| Content | Left: "Agent Sessions" label (text-caption font-semibold). Center: summary counts badge group. Right: collapse chevron button (ChevronDown icon, ghost variant, icon size) |
| Summary counts | Badge group showing: "{N} running" (accent variant), "{N} idle" (success variant), displayed in header right of label |
| Padding | px-3 |

### Table Structure (CONTEXT.md D-04: "name, workbench, status, elapsed time")

| Column | Width | Content | Sortable |
|--------|-------|---------|----------|
| Agent | flex-1 (min 120px) | Workbench icon (lucide, 16px) + workbench name (capitalized) | Yes (alphabetical) |
| Status | 100px | Status dot (8px circle) + status text (capitalized) | Yes (by status priority: running > connecting > idle > suspended > error > stopped) |
| Elapsed | 80px | Elapsed time since session started, font-mono. Format: "12s", "1m 23s", "2h 10m", "--" for stopped | No |
| Actions | 80px | Icon buttons: Play (start) / Square (stop) -- ghost variant, icon size (h-8 w-8) | No |

### Table Row Behavior

| Interaction | Behavior |
|-------------|----------|
| Hover | Background transitions to surface-2. Row border becomes border-strong. |
| Click row | No action (status panel is display-only; actions via buttons) |
| Start button | Visible when session is stopped or suspended. Starts or resumes the session for that workbench (CONTEXT.md D-06: "Force-start spins up a session"). |
| Stop button | Visible when session is running, idle, or connecting. Stops the session immediately with no grace period (CONTEXT.md D-06: "Force-stop kills a session immediately"). |
| Start button disabled | When session is already running, idle, or connecting. Rendered at 50% opacity with disabled cursor. |
| Stop button disabled | When session is already stopped. Rendered at 50% opacity with disabled cursor. |

### Empty Panel State (no sessions created yet)

Centered within the panel area. No table rendered.

### Collapsed State

When collapsed, only the PanelGroup Separator is visible. No panel header or content renders. The separator follows the existing pattern from WorkspaceArea (horizontal drag handle with rounded pill indicator).

### Summary in Collapsed Separator

The separator between content area and status panel includes a minimal inline summary when the panel is collapsed: a small text "{N} agents active" displayed centered in the separator bar, text-text-muted, text-caption. This provides at-a-glance status without opening the panel.

## Toast Notification Contract (CONTEXT.md D-07)

### Library

sonner 2.0.7 -- lightweight, accessible, auto-dismissing toast system.

### Toaster Placement

Bottom-right of the viewport. Uses sonner's `<Toaster />` component mounted in AppShell.

### Toast Theme

Custom theme matching the design system:

| Property | Value |
|----------|-------|
| Background | var(--color-surface-3) |
| Border | 1px solid var(--color-border-strong) |
| Border radius | radius-md (8px) |
| Text color | var(--color-text) |
| Description color | var(--color-text-dim) |
| Shadow | shadow-md |
| Font | var(--font-sans) |
| Width | 360px |

### Toast Events

| Event | Toast Type | Title | Description | Auto-dismiss |
|-------|-----------|-------|-------------|-------------|
| Session connected | success | "{Workbench} agent connected" | "Session resumed successfully." or "New session started." | 4s |
| Session disconnected | warning | "{Workbench} agent disconnected" | "Reconnecting automatically..." | Persistent until reconnected |
| Session error | error | "{Workbench} agent error" | "{error message, truncated to 120 chars}" -- fallback if error message is null or empty: "An unexpected error occurred. Check session logs for details." | 8s |
| Session reconnected | success | "{Workbench} agent reconnected" | "Session restored." | 3s |
| Agent task completed | info | "{Workbench} task complete" | "{brief description of completed work}" | 5s |
| Session force-stopped | info | "{Workbench} agent stopped" | "Session terminated by user." | 3s |

### Toast Deduplication

If the same workbench fires the same event type within 2 seconds, deduplicate (do not stack identical toasts). sonner supports this via the `id` parameter.

## Animation Contract

All animations respect `prefers-reduced-motion: reduce` (durations set to 0ms via existing theme.css).

### Status Panel Expand/Collapse

| Direction | Animation | Duration | Easing |
|-----------|-----------|----------|--------|
| Expand | Panel height grows via react-resizable-panels built-in resize behavior | Instant (handled by library) | N/A |
| Collapse | Panel height shrinks to 0% via react-resizable-panels | Instant (handled by library) | N/A |

No custom animation -- react-resizable-panels handles the resize transition. This matches existing sidebar and pipeline panel behavior.

### Status Dot Pulse

Running sessions show a pulsing dot:

```css
@keyframes session-pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.4; }
}
```

- Applied to the 8px status dot when status is "running" or "connecting"
- Duration: 1500ms, ease-in-out, infinite
- Reduced motion: pulse disabled, static dot at full opacity

### Elapsed Time Counter

- Running sessions update elapsed time every 1 second via `setInterval`
- Uses the existing `useElapsedTime` hook pattern from Phase 5
- Counter starts from session creation timestamp
- Format: "12s" (< 60s), "1m 23s" (< 60m), "2h 10m" (>= 60m)
- Stopped/suspended sessions show final elapsed duration (static)
- Sessions with no start time show "--"

### Toast Enter/Exit

Handled by sonner's built-in animations. No custom override needed. sonner uses slide-up entrance and fade-out exit, which aligns with the design system's ease-out/ease-in pattern.

## Interaction Contract

### Status Panel Toggle

| Interaction | Behavior |
|-------------|----------|
| Click separator drag handle | Expand/collapse via react-resizable-panels drag |
| Keyboard shortcut | Ctrl+Shift+S (toggles status panel visibility via uiStore) |
| First session event | If panel is collapsed and a session connects, auto-expand the panel |
| All sessions stopped | Panel remains in its current state (do not auto-collapse) |

### Session Start/Stop (CONTEXT.md D-06)

| Interaction | Behavior |
|-------------|----------|
| Click Start (Play icon) | Sends WebSocket message `session:start` with workbenchId. Button shows Loader2 spinner while connecting. Transitions to idle/running once connected. |
| Click Stop (Square icon) | Opens a confirmation tooltip: "Stop {Workbench} session? This ends the session immediately." with "Stop Session" (destructive variant, sm size) and "Keep Session" (ghost variant, sm size) buttons. On confirm: sends `session:stop` with workbenchId. |
| Force-start from non-active workbench | Clicking Start for a workbench that is not currently active starts its session without switching the user's view (D-06). |
| Grace period visible | Sessions in grace period (after workbench switch) show "suspended" status with a countdown or just the "suspended" label. |

### Workbench Switching Integration (CONTEXT.md D-01, D-02)

| Behavior | Implementation |
|----------|---------------|
| Switch to workbench with no session | Status shows "stopped" for that workbench. No auto-start (user starts manually or chat panel initiates). |
| Switch away from active workbench | Previous workbench session enters grace period (~30s, exact duration per Claude's discretion). Status shows "suspended" with grace period indicator. |
| Grace period expires | Session status transitions from "suspended" to "stopped". Toast: "{Workbench} session suspended" (info, 3s). |
| Switch back during grace period | Session reactivates immediately. Status transitions from "suspended" to "idle" or "running". No toast (seamless). |

### Keyboard Navigation

| Key | Behavior |
|-----|----------|
| Tab | Moves focus through table rows, then action buttons within each row |
| Enter / Space on Start button | Triggers session start |
| Enter / Space on Stop button | Opens stop confirmation |
| Escape | Closes stop confirmation tooltip |
| Ctrl+Shift+S | Toggle status panel expand/collapse |

## Session Store Contract (Zustand)

New `sessionStore.ts` managing per-workbench session state:

```typescript
type SessionStatus = 'stopped' | 'connecting' | 'idle' | 'running' | 'suspended' | 'error';

interface WorkbenchSession {
  workbenchId: WorkbenchId;
  sessionId: string | null;
  status: SessionStatus;
  startedAt: string | null;     // ISO timestamp
  lastActivity: string | null;  // ISO timestamp of last message
  error: string | null;         // Error message if status === 'error'
  elapsedMs: number | null;     // Total session elapsed time
}

interface SessionState {
  sessions: Map<WorkbenchId, WorkbenchSession>;
  getSession: (id: WorkbenchId) => WorkbenchSession | undefined;
  updateSession: (id: WorkbenchId, update: Partial<WorkbenchSession>) => void;
  setStatus: (id: WorkbenchId, status: SessionStatus) => void;
  // Panel visibility (delegated from uiStore extension)
  statusPanelCollapsed: boolean;
  setStatusPanelCollapsed: (collapsed: boolean) => void;
  toggleStatusPanel: () => void;
}
```

### WebSocket Event Mapping

| WebSocket Event | Store Action |
|----------------|--------------|
| session:connected | Set status to "idle", store sessionId, set startedAt |
| session:disconnected | Set status to "error" (with auto-reconnect), set error message |
| session:reconnected | Clear error, set status to "idle" |
| session:running | Set status to "running" (agent is actively processing) |
| session:idle | Set status to "idle" (agent finished processing, session alive) |
| session:suspended | Set status to "suspended" (grace period started) |
| session:stopped | Set status to "stopped", clear sessionId |
| session:error | Set status to "error", set error message |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Focus visible | All interactive elements (buttons, rows) receive 2px solid accent outline + glow on keyboard focus |
| Keyboard navigation | Tab moves through table rows and action buttons. Enter/Space activates buttons. |
| Screen reader | Panel has role="region" with aria-label="Agent session status". Table uses semantic `<table>` with `<thead>` and `<tbody>`. Each row has aria-label: "{Workbench} agent, status: {status}". Action buttons have aria-label: "Start {Workbench} session" / "Stop {Workbench} session". |
| Reduced motion | Status dot pulse disabled. Toast animations disabled (sonner respects prefers-reduced-motion). |
| Touch targets | All action buttons meet 32px minimum (icon button size h-8 w-8 = 32px). Table rows are 40px height. |
| Color independence | Status communicated through icon shape (Play, Square, Pause, AlertCircle, Loader2, Circle) AND color AND text label -- not color alone. |
| Live region | Panel summary count badge group is an aria-live="polite" region so screen readers announce session count changes. |
| Toast accessibility | sonner provides built-in ARIA live region announcements for toasts. |

## Performance Contract

| Concern | Mitigation |
|---------|-----------|
| Re-render isolation | Session state in dedicated zustand store (sessionStore), not uiStore. Table rows subscribe only to their own workbench's session data via selector: `useSessionStore(s => s.sessions.get(workbenchId))`. |
| Elapsed time updates | Only running sessions tick (1s interval). Stopped/suspended sessions show static value. `useElapsedTime` hook (existing from Phase 5) prevents unnecessary renders. |
| Toast deduplication | Use sonner's `id` parameter (workbenchId + event type) to prevent stacking identical toasts within 2s window. |
| Collapsed panel | When panel is collapsed (0% size), no table rows render. React-resizable-panels unmounts collapsed content. |
| WebSocket batching | Session status updates are low-frequency (seconds, not milliseconds). No RAF batching needed (unlike pipeline events). Direct store updates are sufficient. |

1. Create `packages/backend/src/services/sessionStore.ts` — Backend in-memory session state:

   ```typescript
   /**
    * Backend in-memory store mapping WorkbenchId to session metadata.
    * This is the backend's source of truth for which sessions are active.
    * NOT the same as the frontend zustand sessionStore.
    */
   import type { SessionStatus } from '@afw/shared';

   export interface ManagedSession {
     workbenchId: string;
     sessionId: string | null;
     status: SessionStatus;
     startedAt: string | null;
     lastActivity: string | null;
     error: string | null;
     graceTimeout: ReturnType<typeof setTimeout> | null;
     abortController: AbortController | null;
   }

   export class BackendSessionStore {
     private sessions = new Map<string, ManagedSession>();

     get(workbenchId: string): ManagedSession | undefined {
       return this.sessions.get(workbenchId);
     }

     set(workbenchId: string, session: ManagedSession): void {
       this.sessions.set(workbenchId, session);
     }

     getAll(): Map<string, ManagedSession> {
       return new Map(this.sessions);
     }

     getOrCreate(workbenchId: string): ManagedSession {
       let session = this.sessions.get(workbenchId);
       if (!session) {
         session = {
           workbenchId,
           sessionId: null,
           status: 'stopped',
           startedAt: null,
           lastActivity: null,
           error: null,
           graceTimeout: null,
           abortController: null,
         };
         this.sessions.set(workbenchId, session);
       }
       return session;
     }

     clearGraceTimeout(workbenchId: string): void {
       const session = this.sessions.get(workbenchId);
       if (session?.graceTimeout) {
         clearTimeout(session.graceTimeout);
         session.graceTimeout = null;
       }
     }

     clearAllGraceTimeouts(): void {
       for (const session of this.sessions.values()) {
         if (session.graceTimeout) {
           clearTimeout(session.graceTimeout);
           session.graceTimeout = null;
         }
       }
     }

     /** Count sessions with a given status */
     countByStatus(status: SessionStatus): number {
       let count = 0;
       for (const s of this.sessions.values()) {
         if (s.status === status) count++;
       }
       return count;
     }
   }

   export const backendSessionStore = new BackendSessionStore();
   ```

2. Create `packages/backend/src/services/sessionManager.ts` — Core SessionManager:

   The SessionManager class must:
   - Import `query`, `startup`, `listSessions`, `getSessionMessages`, `getSessionInfo`, `forkSession` from `@anthropic-ai/claude-agent-sdk`
   - Import `type Query` and `type SDKMessage` from the SDK
   - Import `WebSocketHub` from `../ws/hub.js`
   - Import `BackendSessionStore`, `backendSessionStore`, `ManagedSession` from `./sessionStore.js`
   - Import `SessionStatusEvent`, `WORKBENCH_PERSONALITIES`, `SYSTEM_CHANNEL` from `@afw/shared`
   - Use `[SessionManager]` log prefix per project convention

   Implement these methods on the `SessionManager` class:

   **`constructor(hub: WebSocketHub, store: BackendSessionStore, projectDir: string)`**
   - Store hub, store, and projectDir references
   - Set `preWarmed = false`
   - Set `GRACE_PERIOD_MS = 30_000` as a class constant

   **`async initialize(): Promise<void>`**
   - Call `await startup()` to pre-warm CLI (20x faster first query per research)
   - Set `preWarmed = true`
   - Log `[SessionManager] Pre-warmed Agent SDK`
   - Wrap in try/catch: on error, log `[SessionManager] Pre-warm failed: {error}` but do NOT throw (graceful degradation)

   **`async startSession(workbenchId: string): Promise<void>`**
   - Get or create session from store
   - If session status is `idle` or `running`, return early (already active)
   - Clear any existing grace timeout for this workbench
   - Create a new `AbortController`
   - Set status to `connecting`, broadcast via `broadcastStatus()`
   - Get stored sessionId (for resume per SESSION-03)
   - Get personality prompt from `WORKBENCH_PERSONALITIES[workbenchId]` (per SESSION-08, D from CONTEXT)
   - Create an async iterable input stream (for multi-turn via `streamInput` pattern from RESEARCH.md Pattern 2):
     - Use a queue array + resolver pattern: `inputQueue: Array<{type:'user', message:{role:'user', content:[{type:'text', text:string}]}, session_id:string|null, parent_tool_use_id:null}>`
     - `async function* createInputStream()` yields from queue, awaits when empty
     - Store `sendMessage` function on the managed session object (add to ManagedSession interface: `sendMessage?: (text: string) => void`)
   - Call `query({ prompt: createInputStream(), options: { resume: existingSessionId ?? undefined, cwd: projectDir, abortController, persistSession: true, settingSources: ['user', 'project', 'local'], systemPrompt: personality, stderr: (data) => console.error('[SessionManager]', workbenchId, data) } })`
   - Store the Query reference on the managed session
   - Call `consumeStream(workbenchId, queryInstance)` in the background (do NOT await — it runs for the session lifetime)
   - Update store: set abortController, startedAt to `new Date().toISOString()`, status to `idle`
   - Broadcast status `idle` (or `connected` event type)

   **`private async consumeStream(workbenchId: string, session: Query): Promise<void>`**
   - Use `for await (const message of session)` pattern
   - On first message, extract `message.session_id` and store it on the managed session (for resume persistence per SESSION-03)
   - For each message, broadcast it via hub as a WSEnvelope on `_system` channel with type `session:message` and payload `{ workbenchId, message }`
   - Update `lastActivity` on the managed session
   - If message has a `type` field:
     - `result`: broadcast `session:idle` (agent finished processing)
     - Messages indicating tool use: broadcast `session:running`
   - Wrap the entire loop in try/catch:
     - On `AbortError` or signal aborted: expected (session was stopped/suspended), log debug
     - On other errors: set status to `error`, store error message, broadcast `session:error`
   - After loop ends (generator done): if not already stopped/suspended, set status to `stopped`

   **`async suspendSession(workbenchId: string): Promise<void>`**
   - Get session from store. If no session or no active query, return
   - Call `session.abortController?.abort()` to stop the stream
   - Set query reference to null
   - Set status to `suspended`
   - Broadcast status

   **`async stopSession(workbenchId: string): Promise<void>`**
   - Get session from store. If no session, return
   - Clear any grace timeout
   - Call `session.abortController?.abort()` to stop
   - Set query/abortController to null
   - Set status to `stopped`, clear error
   - Broadcast status
   - Do NOT clear sessionId (preserved for resume on next start per SESSION-03)

   **`handleWorkbenchSwitch(newWorkbenchId: string, previousWorkbenchId: string): void`** (per D-01, D-02)
   - Clear any existing grace timeout for `newWorkbenchId` (user switched back during grace period)
   - If `newWorkbenchId` had a `suspended` status and a stored sessionId, call `startSession(newWorkbenchId)` to reactivate
   - For `previousWorkbenchId`: if status is `idle` or `running`:
     - Broadcast status `suspended` immediately
     - Set a grace timeout of `GRACE_PERIOD_MS`
     - When timeout fires: call `suspendSession(previousWorkbenchId)`
     - Store the timeout handle on the managed session

   **`async forkSession(workbenchId: string): Promise<string | null>`** (per SESSION-09)
   - Get session from store. If no sessionId, return null
   - Call Agent SDK `forkSession` option on a new `query()` call
   - Return the new session ID
   - Log the fork

   **`async getSessionHistory(workbenchId: string): Promise<unknown[]>`** (per SESSION-07)
   - Get session from store. If no sessionId, return []
   - Call `getSessionMessages(sessionId, { dir: projectDir, limit: 50 })`
   - Return the messages array

   **`async listWorkbenchSessions(): Promise<unknown[]>`** (per SESSION-07)
   - Call `listSessions({ dir: projectDir, limit: 20 })`
   - Return results

   **`private broadcastStatus(workbenchId: string, status: SessionStatus, extra?: Partial<SessionStatusEvent>): void`**
   - Get session from store
   - Build a `SessionStatusEvent` payload: `{ workbenchId, sessionId: session?.sessionId ?? null, status, startedAt: session?.startedAt ?? null, lastActivity: session?.lastActivity ?? null, error: session?.error ?? null, ...extra }`
   - Build a `WSEnvelope`: `{ channel: '_system', type: 'session:status', payload, ts: new Date().toISOString() }`
   - Call `this.hub.broadcastAll(JSON.stringify(envelope))`

   **`async shutdown(): Promise<void>`**
   - For all sessions in store: abort and clean up
   - Clear all grace timeouts

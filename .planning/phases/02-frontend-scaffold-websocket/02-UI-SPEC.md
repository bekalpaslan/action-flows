## Component Inventory (Phase 2 Scope)

Phase 2 builds structural containers, not design system components. These are plain HTML elements styled with existing CSS tokens. The Phase 3 component library will replace them.

| Component | Element | Purpose | Styling Approach |
|-----------|---------|---------|------------------|
| `AppShell` | `<div>` | Root layout container, 3-region grid | CSS custom properties from `design-tokens.css` |
| `SidebarPlaceholder` | `<nav>` + `<ul>` | 7 workbench nav items | Inline token references (`--app-bg-secondary`, `--text-secondary`) |
| `WorkspaceArea` | `<main>` | Active workbench content area | `--app-bg-primary` background |
| `ChatPlaceholder` | `<aside>` | Right panel placeholder for Phase 7 chat | `--app-bg-secondary` background, "Chat panel -- Phase 7" placeholder text |
| `WebSocketStatus` | `<span>` | Connection indicator in shell header/footer | Status color tokens + 8px dot |

**Layout contract for the shell:**
- Sidebar: fixed 220px width, left edge
- Workspace: fluid, fills remaining space
- Chat placeholder: fixed 300px width, right edge
- No resize handles in Phase 2 (react-resizable-panels arrives in Phase 4)
- Minimum viewport: 1024px width (Electron desktop app)

**Source:** CONTEXT.md "Minimal app shell -- sidebar placeholder + workspace + chat placeholder." Layout percentages from REQUIREMENTS.md LAYOUT-01 converted to fixed widths for the pre-resize scaffold.

## Interaction States

Phase 2 has minimal interaction. The following states must be implemented:

| Interaction | State | Visual |
|-------------|-------|--------|
| Sidebar item default | Unselected | `--text-secondary` text, transparent background |
| Sidebar item hover | Hovered | `--app-bg-tertiary` background, `--text-primary` text |
| Sidebar item active | Selected workbench | `--system-blue` left border (3px), `--text-primary` text, `--app-bg-tertiary` background |
| WebSocket connected | Stable | 8px circle filled `--status-success-text`, "Connected" label |
| WebSocket reconnecting | Unstable | 8px circle filled `--status-warning-text`, "Reconnecting..." label, pulse animation at `--duration-slow` (600ms) |
| WebSocket disconnected | Offline | 8px circle filled `--status-error-text`, "Disconnected" label |

Transition for sidebar hover: `background-color 150ms ease`.

**Source:** Interaction patterns derived from FOUND-04 success criteria (workbench shell renders, switching workbenches works).

## Phase Boundary Notes

This UI-SPEC is intentionally minimal because Phase 2 is a structural scaffold:

- **Phase 3** (Design System) will introduce the full component library, Tailwind v4 migration, CVA variants, and the machine-readable component manifest. That phase will have its own comprehensive UI-SPEC.
- **Phase 4** (Layout & Navigation) will introduce react-resizable-panels, proper panel constraints, command palette, and the polished 3-panel layout. It will also have its own UI-SPEC.
- **Phase 2 success** = the shell renders, workbench switching works, WebSocket multiplexing is live, and the build pipeline produces a working Electron app. Visual polish is explicitly out of scope.

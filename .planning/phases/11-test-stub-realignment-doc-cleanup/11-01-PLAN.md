**Root cause of getActiveCircuitBreakers test failure:**
The test's mock storage uses `pattern.replace('*', '')` which only replaces the FIRST asterisk. When the service calls `storage.keys('healingQuota:*:*:2026-04-05')`, the mock converts it to prefix `'healingQuota::*:2026-04-05'` (one `*` remains literal), then filters keys starting with that prefix. Real stored keys look like `'healingQuota:work:build-flow:2026-04-05'` — none match the mangled prefix, so the mock returns `[]` and the test sees `breakers.length=0` instead of 1.

This is a TEST BUG, not a service bug. Fix the mock's `keys` implementation (inside the test file) to handle multi-wildcard patterns. The production Storage implementations (Memory/Redis) handle multi-wildcard patterns correctly — only the test mock is broken.

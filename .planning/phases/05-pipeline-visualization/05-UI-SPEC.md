# Phase 5 -- UI Design Contract

> Visual and interaction contract for the pipeline visualization phase. Generated by gsd-ui-researcher.
>
> This phase replaces PipelinePlaceholder.tsx with a live, interactive horizontal node-based pipeline powered by @xyflow/react v12. All visual values inherit from the Phase 3 design system (theme.css tokens, component library). This contract defines the pipeline-specific visual elements: custom node rendering, status states, animations, edge styling, drawer overlay, and empty states.

---

## Color

Inherited from Phase 3 (03-UI-SPEC.md). Pipeline-specific color applications:

### Node Status Colors

| Status | Background | Border | Icon | Badge Variant |
|--------|-----------|--------|------|---------------|
| pending | var(--color-surface-2) | var(--color-border) | Circle (text-text-muted) | default |
| running | var(--color-surface-2) | var(--color-accent) | Loader2 (text-accent, spinning) | accent |
| complete | var(--color-surface-2) | var(--color-success) | CheckCircle2 (text-success) | success |
| failed | var(--color-surface-2) | var(--color-destructive) | XCircle (text-destructive) | error |
| skipped | var(--color-surface-2) | var(--color-border) | SkipForward (text-text-muted) | default |

All nodes use surface-2 background to maintain the 60/30/10 ratio. Status is communicated through border color (2px left border) and icon color -- not through background fills. This keeps the pipeline visually coherent when multiple statuses are visible simultaneously.

### Edge Colors

| Context | Color | Width |
|---------|-------|-------|
| Default edge | var(--color-border-strong) | 2px |
| Active path (running) | var(--color-accent) | 2px |
| Completed path | var(--color-success) at 50% opacity | 2px |
| Failed path | var(--color-destructive) at 50% opacity | 2px |

### Drawer Colors

| Element | Color |
|---------|-------|
| Drawer background | var(--color-surface-3) |
| Drawer border | var(--color-border-strong) |
| Drawer overlay scrim | rgba(0, 0, 0, 0.3) |

### Decision Gate (Diamond) Colors

Decision gates follow the same status color mapping as step nodes. The diamond shape itself is the visual differentiator -- no additional color treatment needed.

**Accent usage in this phase (additions to Phase 3 exhaustive list):**
10. Active pipeline node border (running status)
11. Active edge path stroke (data flowing through edge)
12. Selected node highlight ring (click-to-inspect)

## Node Design Contract

### StepNode (Rounded Rectangle)

| Property | Value |
|----------|-------|
| Dimensions | min-width: 160px, height: auto (content-driven, typically 72-88px) |
| Border radius | radius-md (8px) |
| Background | var(--color-surface-2) |
| Border | 1px solid {status-color}, plus 3px left accent bar in status color |
| Shadow | shadow-sm at rest, shadow-md on hover |
| Layout | Vertical stack: [status icon + step name] row, [elapsed time] row, [agent model] row |
| Status icon | 16px lucide icon, left-aligned in first row |
| Step name | 13px semibold, truncated with ellipsis at 120px max-width |
| Elapsed time | 13px normal, text-text-dim, format: "12s" or "1m 23s" or "--" if pending |
| Agent model | 13px normal, text-text-muted, truncated (e.g. "opus-4", "sonnet-4") |
| Hover | border transitions to border-strong, shadow elevates to shadow-md, cursor: pointer |
| Selected (clicked) | 2px solid accent ring (same as focus-visible), shadow-glow-focus |
| Handle positions | Source: right center. Target: left center. |

### GateNode (Diamond)

| Property | Value |
|----------|-------|
| Dimensions | 64px x 64px (square, rotated 45deg via CSS transform) |
| Inner content | Rotated -45deg to keep text upright |
| Border radius | radius-xs (4px) on the rotated square |
| Background | var(--color-surface-2) |
| Border | 2px solid {status-color} |
| Shadow | shadow-sm at rest |
| Content | Single icon: GitBranch (lucide) at 20px, centered |
| Label | Rendered below the diamond (not inside), 13px, text-text-dim, max 80px truncated |
| Hover | Same as StepNode hover |
| Selected | Same as StepNode selected |
| Handle positions | Source: right center. Target: left center. (Adjusted for diamond geometry via CSS offset) |

### Fork/Merge Visual Pattern

| Pattern | Rendering |
|---------|-----------|
| Fork (1-to-N) | Single source node with N outgoing edges, each targeting a separate node. Dagre handles vertical fan-out automatically with rankdir: 'LR'. |
| Merge (N-to-1) | N source nodes each with an edge targeting the same merge node. Dagre collapses the fan-in. |
| Edge routing | Smoothstep edge type from @xyflow/react for clean right-angle routing around forked paths. |
| Fork source marker | No special marker. The visual fan-out is self-explanatory. |
| Merge target marker | No special marker. The visual fan-in is self-explanatory. |

## Animation Contract

All animations respect `prefers-reduced-motion: reduce` (durations set to 0ms via existing theme.css).

### Node Status Transitions

| Transition | Animation | Duration | Easing |
|------------|-----------|----------|--------|
| pending -> running | Left border fades to accent color + pulse glow | 300ms | ease-in-out |
| running -> complete | Left border fades to success, pulse stops, checkmark fades in | 300ms | ease-out |
| running -> failed | Left border fades to destructive, X icon fades in with subtle shake | 300ms | ease-out |
| any -> skipped | Border fades to default, skip icon fades in | 200ms | ease-out |

### Running Node Pulse

```css
@keyframes pipeline-pulse {
  0%, 100% { box-shadow: 0 0 0 0 rgba(62, 103, 191, 0); }
  50% { box-shadow: 0 0 8px 2px rgba(62, 103, 191, 0.15); }
}
```

- Applied to nodes with status "running"
- Duration: 2000ms, infinite loop
- Uses accent color (--color-accent: #3e67bf) at low opacity
- Reduced motion: pulse disabled, static glow-default shadow instead

### Edge Flow Animation

Active edges (connecting to a running node) use a CSS animated dash pattern:

```css
@keyframes pipeline-edge-flow {
  from { stroke-dashoffset: 24; }
  to { stroke-dashoffset: 0; }
}
```

- stroke-dasharray: 8 4
- Duration: 1000ms, linear, infinite
- Applied only to edges whose target node has status "running"
- Reduced motion: static dashed line, no animation

### Drawer Enter/Exit

| Direction | Animation | Duration | Easing |
|-----------|-----------|----------|--------|
| Enter (open) | Slide from right + fade in | 200ms (--duration-normal) | ease-out |
| Exit (close) | Slide to right + fade out | 150ms (--duration-fast) | ease-in |

## Interaction Contract

### Click-to-Inspect (CONTEXT.md D-03)

| Interaction | Behavior |
|-------------|----------|
| Click on node | Opens the detail drawer over the pipeline region (right side). Does NOT navigate or affect the content area below. Sets the node as "selected" with accent ring. |
| Click same node again | Closes the drawer. Deselects the node. |
| Click different node while drawer open | Drawer content updates to the new node. Previous node deselected, new node selected. |
| Click canvas background | Closes the drawer. Deselects all nodes. |
| Escape key | Closes the drawer. Deselects all nodes. |
| Pan/zoom | Standard @xyflow/react pan (mouse drag on canvas) and zoom (scroll wheel). Does not close the drawer. |

### Horizontal Scroll (PIPE-05)

| Behavior | Implementation |
|----------|---------------|
| Long pipelines | @xyflow/react's built-in viewport with fitView on initial load. User pans freely. |
| Fit view | Auto-fit on chain load. Button in top-right corner of pipeline region: "Fit" (lucide Maximize2 icon, 32px, ghost variant). |
| Mini-map | Disabled for Phase 5. Pipeline region is already compact (~25% of workspace height). Mini-map adds visual clutter without benefit at this scale. |
| Zoom constraints | minZoom: 0.25, maxZoom: 2.0 |

### Workbench Switching (PIPE-07 / D-06)

| Behavior | Implementation |
|----------|---------------|
| Switch workbench | Pipeline re-renders with the new workbench's chain data from pipelineStore. Drawer closes. Viewport resets to fitView. |
| No active chain | Empty state renders (see Copywriting Contract). |
| Chain completes | Pipeline remains visible showing final state for 5 seconds, then transitions to empty state. Completed chain data moves to history (accessible from content area, not pipeline region). |

The drawer slides in from the right edge of the pipeline region, overlaying the pipeline canvas. It does NOT extend beyond the pipeline region boundary.

### Drawer Layout

| Property | Value |
|----------|-------|
| Width | 320px fixed |
| Max height | 100% of pipeline region height |
| Position | Absolute, right: 0, top: 0, bottom: 0 within the pipeline container |
| Background | var(--color-surface-3) |
| Border | 1px solid var(--color-border-strong) on the left edge |

| Shadow | shadow-lg on the left side |

| Section | Content |
|---------|---------|

## Pipeline Data Contract

The pipeline store (zustand) holds per-workbench state. This defines the data shapes the UI renders from.

### Node Data Shape (for @xyflow/react Node.data)

```typescript
interface StepNodeData {
  type: 'step';
  stepNumber: number;
  name: string;           // Step name / action
  status: 'pending' | 'running' | 'complete' | 'failed' | 'skipped';
  elapsedMs: number | null;
  agentModel: string | null;
  description: string | null;
  inputs: Record<string, unknown> | null;
  result: unknown | null;
  error: string | null;
  suggestion: string | null;
  fileChanges: Array<{ path: string; type: 'created' | 'modified' | 'deleted' }> | null;
  startedAt: string | null;  // ISO timestamp
}

interface GateNodeData {
  type: 'gate';
  label: string;
  status: 'pending' | 'running' | 'complete' | 'failed';
  passCount: number;
  failCount: number;
  outcome: string | null;  // Which path was taken
}
```

### WebSocket Event Mapping

| WebSocket Event | Pipeline Store Action |
|----------------|----------------------|
| chain:compiled | Initialize nodes + edges from ChainStepSnapshot[]. All nodes set to 'pending'. Run dagre layout. fitView. |
| chain:started | No visual change (nodes already pending from compiled). |
| step:spawned | Update node data with action, model, inputs, description. |
| step:started | Set node status to 'running'. Start elapsed time counter. Set incoming edge to active. |
| step:completed | Set node status to 'complete'. Stop elapsed timer. Set final duration. Update result, fileChanges. Set edge to completed. |
| step:failed | Set node status to 'failed'. Stop elapsed timer. Set error, suggestion. Set edge to failed. |
| step:skipped | Set node status to 'skipped'. Set reason. |
| chain:completed | Mark all edges as their final state. Start 5-second display timer before transitioning to empty state. |

## Performance Contract (from PITFALLS.md P7)

| Concern | Mitigation |
|---------|-----------|
| Re-render cascade | Node/edge state in zustand, NOT React useState. Selector pattern: each node component selects only its own data via `useStore(s => s.nodes.find(n => n.id === id))`. |
| Handler memoization | All onNodesChange, onEdgesChange, onNodeClick handlers wrapped in useCallback with stable deps. |
| Custom node declaration | StepNode and GateNode components defined outside the pipeline parent component (module-level), never inline. |
| nodeTypes object | Defined as a module-level constant: `const nodeTypes = { step: StepNode, gate: GateNode }` -- never recreated on render. |
| WebSocket batching | Collect incoming pipeline events for 100ms before applying to store (requestAnimationFrame debounce). |
| Large chains (>50 nodes) | Not expected in Phase 5 (active chain only, typical chains are 3-15 steps). No viewport culling needed yet. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Focus visible | Nodes receive 2px solid accent outline + glow on keyboard focus (Tab navigation through nodes) |
| Keyboard navigation | Tab moves between nodes in step order. Enter opens drawer for focused node. Escape closes drawer. |
| Screen reader | Each node has aria-label: "{step name}, status: {status}, step {N} of {total}". Drawer has role="dialog" with aria-labelledby pointing to the step name heading. |
| Reduced motion | All pulse/flow animations disabled. Static visual indicators only. Drawer transitions instant (0ms). |
| Touch targets | Nodes meet 44px minimum hit area (StepNode: 160x72+ px, GateNode: 64x64 px -- both exceed minimum). |
| Color independence | Status is communicated through icon shape AND color (CheckCircle2, XCircle, Loader2, SkipForward, Circle) -- not color alone. |
| Drawer focus trap | When drawer opens, focus moves to the close button. Tab cycles within drawer content. Escape returns focus to the triggering node. |

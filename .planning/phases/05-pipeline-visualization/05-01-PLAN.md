**Step 4 -- Create pipeline types** at `packages/app/src/lib/pipeline-types.ts`:
```typescript
import type { Node, Edge } from '@xyflow/react';
import type { WorkbenchId } from './types';

export type NodeStatus = 'pending' | 'running' | 'complete' | 'failed' | 'skipped';

export type StepNodeData = {
  type: 'step';
  stepNumber: number;
  name: string;
  status: NodeStatus;
  elapsedMs: number | null;
  agentModel: string | null;
  description: string | null;
  inputs: Record<string, unknown> | null;
  result: unknown | null;
  error: string | null;
  suggestion: string | null;
  fileChanges: Array<{ path: string; type: 'created' | 'modified' | 'deleted' }> | null;
  startedAt: string | null;
};

export type GateNodeData = {
  type: 'gate';
  label: string;
  status: NodeStatus;
  passCount: number;
  failCount: number;
  outcome: string | null;
};

export type PipelineNodeData = StepNodeData | GateNodeData;

export type StepNodeType = Node<StepNodeData, 'step'>;
export type GateNodeType = Node<GateNodeData, 'gate'>;
export type PipelineNode = Node<PipelineNodeData>;

export interface PipelineState {
  nodes: PipelineNode[];
  edges: Edge[];
  selectedNodeId: string | null;
  drawerOpen: boolean;
  chainId: string | null;
}

export function createEmptyPipelineState(): PipelineState {
  return { nodes: [], edges: [], selectedNodeId: null, drawerOpen: false, chainId: null };
}
```

**Pipeline Layout Utility** at `packages/app/src/lib/pipeline-layout.ts`:

Import dagre, Node, Edge from @xyflow/react, ChainStepSnapshot from @afw/shared, and types from pipeline-types.ts.

Constants (exported for tests):
- `STEP_WIDTH = 160`
- `STEP_HEIGHT = 88`
- `GATE_SIZE = 64`

Function `isGateStep(step: ChainStepSnapshot): boolean`:
Returns true if `step.action` matches any of: contains "gate", "check", "decision" (case-insensitive). This is the Phase 5 naming convention for gate detection per RESEARCH.md Open Question 1.

Function `layoutPipeline(steps: ChainStepSnapshot[]): { nodes: PipelineNode[]; edges: Edge[] }`:
1. If steps is empty, return `{ nodes: [], edges: [] }`.
2. Create dagre graph: `new dagre.graphlib.Graph().setDefaultEdgeLabel(() => ({}))`
3. Set graph options: `{ rankdir: 'LR', nodesep: 24, ranksep: 48 }`
4. For each step:
   - Determine if gate via `isGateStep(step)`
   - Set node with width/height: gate uses `GATE_SIZE`, step uses `STEP_WIDTH`/`STEP_HEIGHT`
5. For each step, add edges:
   - If `step.waitsFor?.length > 0`: add edge from each dependency to this step
   - Else if index > 0: add sequential edge from previous step
6. Run `dagre.layout(g)`
7. Map steps to Node[] -- CRITICAL: adjust position from dagre center to top-left: `{ x: pos.x - w/2, y: pos.y - h/2 }`
8. Node data: map ChainStepSnapshot to StepNodeData or GateNodeData. All statuses start as 'pending'. elapsedMs, result, error, suggestion, fileChanges, startedAt all null.
9. Build edges with `type: 'smoothstep'` and ids like `e-{source}-{target}`.
10. Return `{ nodes, edges }`.

**Pipeline Store** at `packages/app/src/stores/pipelineStore.ts`:

Create zustand store following exact pattern from uiStore.ts (module-level singleton, no provider).

Interface `PipelineStoreState`:
```typescript
{
  pipelines: Map<WorkbenchId, PipelineState>;
  initChain: (workbenchId: WorkbenchId, chainId: string, nodes: PipelineNode[], edges: Edge[]) => void;
  updateNodeStatus: (workbenchId: WorkbenchId, nodeId: string, data: Partial<StepNodeData> | Partial<GateNodeData>) => void;
  updateEdgeStatus: (workbenchId: WorkbenchId, edgeId: string, data: Partial<Edge>) => void;
  selectNode: (workbenchId: WorkbenchId, nodeId: string | null) => void;
  clearPipeline: (workbenchId: WorkbenchId) => void;
}
```

Actions:
- `initChain`: Creates new PipelineState in the Map for the given workbenchId. Sets nodes, edges, chainId. selectedNodeId = null, drawerOpen = false.
- `updateNodeStatus`: Finds the pipeline for workbenchId, finds the node by id, spreads new data into node.data. Creates new Map/array references (immutable per @xyflow/react v12 requirement).
- `updateEdgeStatus`: Same pattern for edges.
- `selectNode`: If nodeId is non-null, sets selectedNodeId and drawerOpen = true. If null, sets selectedNodeId = null and drawerOpen = false.
- `clearPipeline`: Deletes the workbenchId key from the Map.

Export as `export const usePipelineStore = create<PipelineStoreState>(...)`.

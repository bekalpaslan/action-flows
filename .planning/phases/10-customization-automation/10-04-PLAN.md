Class `HealingQuotaTracker`:

- `private getTodayKey(workbenchId: string, flowId: string): string` — returns `healingQuota:${workbenchId}:${flowId}:${new Date().toISOString().slice(0, 10)}` (YYYY-MM-DD format, daily reset via key change)

- `async isQuotaExhausted(workbenchId: string, flowId: string): Promise<boolean>` — returns `quota.attemptsUsed >= MAX_HEALING_ATTEMPTS_PER_DAY`
- `async getActiveCircuitBreakers(): Promise<HealingQuota[]>` — keys pattern `healingQuota:*:*:${today}`, filter for attemptsUsed >= maxAttempts

**Create packages/backend/src/services/healingService.ts:**

Per RESEARCH.md Pattern 4 (Healing as Approval + Error-Class Match):

```typescript
import type { Storage } from '../storage/index.js';
import type { HealingAttempt, HealingAttemptId, ErrorClass } from '@afw/shared';
import { HEALING_ERROR_CLASSES } from '@afw/shared';
import type { ApprovalService } from './approvalService.js';
import type { HealingQuotaTracker } from './healingQuotaTracker.js';
```

Class `HealingService`:
- Constructor: `approvalService: ApprovalService`, `quotaTracker: HealingQuotaTracker`, `storage: Storage`

- `async onRuntimeError(err: { message: string; errorClass: ErrorClass }, ctx: { workbenchId: string; flowId: string; sessionId: string }): Promise<HealingAttempt | null>`:
  1. Check D-01: if `!HEALING_ERROR_CLASSES.includes(err.errorClass)`, log and return null (contract violations don't trigger healing)
  2. Check D-02: `const quota = await this.quotaTracker.getTodayQuota(ctx.workbenchId, ctx.flowId)`. If `quota.attemptsUsed >= 2`, return null and log "Circuit breaker active for {workbenchId}:{flowId}"
  3. D-03: Create approval request via `this.approvalService.createRequest({ action: 'heal_runtime_error', description: \`Attempt to heal: ${err.message} (attempt ${quota.attemptsUsed + 1}/2 today)\`, workbenchId: ctx.workbenchId, sessionId: ctx.sessionId })`
  4. Create HealingAttempt record: `{ id: crypto.randomUUID() as HealingAttemptId, workbenchId: ctx.workbenchId, flowId: ctx.flowId, errorClass: err.errorClass, errorMessage: err.message, proposedFix: 'Agent will attempt to fix the error', status: 'awaiting_approval', createdAt: new Date().toISOString() }`
  5. Store at `healingAttempt:${attempt.id}` and also correlate with approval ID: `healingApprovalMap:${approval.id}` -> attempt.id
  6. Return the attempt

- `async resolveHealing(approvalId: string, decision: 'approved' | 'declined'): Promise<HealingAttempt | null>`:
  1. Read attempt ID from `healingApprovalMap:${approvalId}`
  2. Read attempt from `healingAttempt:${attemptId}`
  3. If decision is 'approved': increment quota via `quotaTracker.incrementAttempt`, set status to 'approved', (actual fix execution is deferred — for v1 this marks intent; real fix would be dispatched via SessionManager in a future integration)
  4. If decision is 'declined': set status to 'declined'
  5. Set resolvedAt, write back. Return updated attempt.

- `async getHistory(limit?: number): Promise<HealingAttempt[]>` — keys pattern `healingAttempt:*`, parse, sort by createdAt desc, slice to limit (default 50)

- `async getAttemptsByWorkbench(workbenchId: string): Promise<HealingAttempt[]>` — filter history by workbenchId

**Create packages/backend/src/routes/healing.ts:**

Express Router factory: `export default function createHealingRouter(healingService: HealingService, quotaTracker: HealingQuotaTracker): Router`

Routes:
- `GET /quota/:workbenchId/:flowId` — returns `{ success: true, quota: HealingQuota }`
- `GET /history` — query param `?limit=50`. Returns `{ success: true, attempts: HealingAttempt[] }`
- `GET /history/:workbenchId` — returns `{ success: true, attempts: HealingAttempt[] }` for specific workbench
- `POST /attempts` — body: `{ errorMessage, errorClass, workbenchId, flowId, sessionId }`. Calls `healingService.onRuntimeError`. Returns 201 `{ success: true, attempt }` or 409 if circuit breaker active `{ success: false, error: 'Circuit breaker active', quota }`
- `POST /resolve/:approvalId` — body: `{ decision: 'approved' | 'declined' }`. Calls `healingService.resolveHealing`. Returns `{ success: true, attempt }`
- `GET /circuit-breakers` — returns `{ success: true, breakers: HealingQuota[] }` (active circuit breakers)

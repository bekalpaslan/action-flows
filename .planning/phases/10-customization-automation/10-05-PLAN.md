Class `CustomWorkbenchService`:
- Constructor: `private storage: Storage`

Methods:
- `async createWorkbench(data: { name: string; iconName: string; greeting: string; tone: string; systemPromptSnippet: string }): Promise<CustomWorkbench>` — generates ID from name slug (lowercase, replace non-alphanumeric with dash, prepend `custom-`) as `CustomWorkbenchId`. Validates name is non-empty. Checks that generated ID is NOT a default workbench ID via `isDefaultWorkbench(id)`. Checks for name uniqueness by listing all custom workbenches and comparing names case-insensitively. Stores at key `customWorkbench:${id}`. Returns created workbench with createdAt.

- `async listWorkbenches(): Promise<CustomWorkbench[]>` — keys pattern `customWorkbench:*`, parse, return sorted by createdAt asc (oldest first, for stable sidebar order)

- `async getWorkbench(id: CustomWorkbenchId): Promise<CustomWorkbench | null>` — single key lookup

- `async updateWorkbench(id: CustomWorkbenchId, updates: Partial<Pick<CustomWorkbench, 'name' | 'iconName' | 'greeting' | 'tone' | 'systemPromptSnippet'>>): Promise<CustomWorkbench | null>` — D-11 guard: `if (isDefaultWorkbench(id as string)) throw new Error('Cannot modify default workbenches')`. Reads, merges, writes back.

- `async deleteWorkbench(id: CustomWorkbenchId): Promise<boolean>` — D-11 guard: reject if default. Deletes `customWorkbench:${id}`. Also cleans up related data: `skill:${id}:*`, `schedule:${id}:*` keys (cascade delete).

**Create packages/backend/src/routes/customWorkbenches.ts:**

Express Router factory: `export default function createCustomWorkbenchesRouter(workbenchService: CustomWorkbenchService): Router`

Routes:
- `GET /` — returns `{ success: true, workbenches: CustomWorkbench[] }`
- `POST /` — body: `{ name, iconName, greeting, tone, systemPromptSnippet }`. Validates name non-empty. Returns 201 `{ success: true, workbench }`. On name conflict returns 409 `{ success: false, error: 'A workbench with this name already exists.' }`. On default ID collision returns 400.
- `PUT /:id` — partial updates. Returns `{ success: true, workbench }` or 404. Returns 400 if attempting to modify default workbench.
- `DELETE /:id` — returns `{ success: true }` or 404. Returns 400 if attempting to delete default workbench.

**Create packages/backend/src/services/scheduledTaskService.ts:**

```typescript
import { Cron } from 'croner';
import type { Storage } from '../storage/index.js';
import type { ScheduledTask, ScheduledTaskId, TaskRun } from '@afw/shared';
import { MAX_RUN_HISTORY } from '@afw/shared';
```

Class `ScheduledTaskService`:
- Private fields: `storage: Storage`, `jobs: Map<string, Cron>` (maps taskId to Cron instance)
- Constructor receives `storage: Storage`

Methods:
- `async createTask(workbenchId: string, data: { name: string; description: string; cronExpression: string; target: { type: 'flow' | 'action'; name: string } }): Promise<ScheduledTask>` — validates cron by creating a `new Cron(cronExpression)` and catching parse errors (throw descriptive error on invalid cron). Generates UUID as ScheduledTaskId. Creates task object with `createdAt: new Date().toISOString()`, `nextRun: job.nextRun()?.toISOString() ?? null`, `lastStatus: null`, `lastRun: null`. Stores at key `schedule:${workbenchId}:${taskId}`. Registers Cron job via `registerJob(task)`. Returns task.

- `async listTasks(workbenchId: string): Promise<ScheduledTask[]>` — keys pattern `schedule:${workbenchId}:*`, parse each, update `nextRun` from live Cron instance if exists, return sorted by createdAt desc.

- `async getTask(workbenchId: string, taskId: ScheduledTaskId): Promise<ScheduledTask | null>` — single key lookup.

- `async updateTask(workbenchId: string, taskId: ScheduledTaskId, updates: Partial<Pick<ScheduledTask, 'name' | 'description' | 'cronExpression' | 'target'>>): Promise<ScheduledTask | null>` — read, merge, if cronExpression changed: stop old Cron, validate new, register new. Write back. Return updated task.

- `async deleteTask(workbenchId: string, taskId: ScheduledTaskId): Promise<boolean>` — stop Cron job if running, delete task key and run history key, return true.

- `async runNow(workbenchId: string, taskId: ScheduledTaskId): Promise<TaskRun>` — reads task, calls `executeTask(task)` directly. Returns the run record.

- `private async executeTask(task: ScheduledTask): Promise<TaskRun>` — records start time. Try: dispatch action (for v1: log execution + broadcast WS event `schedule:task_executed` with taskId and workbenchId). Catch: log error, do NOT retry (D-09). Record run with `recordRun(task.id, run)`. Update task's `lastStatus` and `lastRun` in storage.

- `private async recordRun(taskId: ScheduledTaskId, run: TaskRun): Promise<void>` — reads existing history from `scheduleRun:${taskId}`, prepends new run, slices to MAX_RUN_HISTORY (10 per D-08), writes back.

- `async getRunHistory(taskId: ScheduledTaskId): Promise<TaskRun[]>` — reads `scheduleRun:${taskId}`, parses JSON, returns array (empty if not found).

- `private registerJob(task: ScheduledTask): void` — stops existing job for taskId if any. Creates `new Cron(task.cronExpression, { timezone: 'UTC' }, async () => { await this.executeTask(task); })`. Stores in `this.jobs`.

- `getNextRun(taskId: string): string | null` — returns `this.jobs.get(taskId)?.nextRun()?.toISOString() ?? null`.

- `async loadAllTasks(): Promise<void>` — called on service initialization. Reads all `schedule:*` keys, registers Cron jobs for each. Checks for missed runs (compare lastRun against previousRun from Cron per RESEARCH.md Pitfall 3).

- `stopAll(): void` — stops all Cron jobs. Called on server shutdown.

**Create packages/backend/src/routes/scheduledTasks.ts:**

Express Router factory: `export default function createScheduledTasksRouter(taskService: ScheduledTaskService): Router`

Routes:
- `GET /:workbenchId` — returns `{ success: true, tasks: ScheduledTask[] }`
- `POST /:workbenchId` — body: `{ name, description, cronExpression, target }`. Validates name non-empty, cronExpression non-empty. Returns 201 `{ success: true, task: ScheduledTask }`. On invalid cron, returns 400 `{ success: false, error: 'Invalid cron expression...' }`
- `PUT /:workbenchId/:taskId` — partial updates. Returns `{ success: true, task }` or 404
- `DELETE /:workbenchId/:taskId` — returns `{ success: true }` or 404
- `POST /:workbenchId/:taskId/run` — manual trigger (D-09). Returns `{ success: true, run: TaskRun }` or 404/500
- `GET /:workbenchId/:taskId/runs` — returns `{ success: true, runs: TaskRun[] }`

Note: CUSTOM-06 has no CONTEXT decisions. This is a lightweight v1 implementation showing learnings from healing history. The component reads from `useHealingStore` to display resolved healing attempts as "learnings" — each successful heal is a learned pattern.

Props: none

Implementation:
- On mount, calls `useHealingStore.getState().loadHistory()`
- Header: heading "Learnings" (text-heading font-semibold), subtitle "Patterns learned from self-healing across workbenches." (text-caption text-text-dim)
- Search input: Input with placeholder "Search learnings..." that filters displayed items by errorClass, errorMessage, or workbenchId (case-insensitive substring match). The filter applies in real-time as the user types (controlled input with `onChange` updating a `searchQuery` state variable).
- Derive learnings from healing attempts: filter to status 'succeeded' OR 'approved' (since v1 may not always have the full outcome transition; show approved attempts as provisional learnings so the list is not empty). Deduplicate by errorClass where appropriate, show error class, error message summary, resolution, and timestamp.
- ContentList: for each learning:
  - ContentListItem with primary = errorClass (Badge), secondary = first 100 chars of errorMessage, metadata = workbenchId + relative timestamp
- StatCard row at top: "Patterns learned" (count of succeeded unique error classes), "Success rate" (from healingStore.getSuccessRate)
- Empty state: "No learnings yet. Self-healing will create learnings as it resolves runtime errors."

Export: `export function LearningsBrowser()`

Note: CUSTOM-07 has no CONTEXT decisions. This is a lightweight v1 read-only panel that displays MCP server information from the backend.

Props: none

Implementation:
- On mount, fetches MCP server info from backend: `GET /api/mcp/servers`
- Header: heading "MCP Servers" (text-heading font-semibold), subtitle "Registered Model Context Protocol servers and their capabilities." (text-caption text-text-dim)
- State: `servers: Array<{ name: string; toolCount: number; status: string }>`, `loading: boolean`, `error: string | null`
- Fetch with try/catch. **Graceful fallback on any error (network failure, 404, 500, etc.):** Display a non-error informational state:
  ```tsx
  <div className="flex flex-col items-center gap-3 py-12 text-center">
    <Server className="h-8 w-8 text-text-muted" />

    <p className="text-caption text-text-dim max-w-md">
      Could not reach the MCP server endpoint. Ensure the backend is running and the MCP
      server package is configured. This panel will populate automatically when MCP servers are available.
    </p>
  </div>
  ```
  Import `Server` icon from `lucide-react`. Do NOT show a red error state — this is expected when MCP is not configured.
- When data is available: render each server as a Card with CardHeader (server name as CardTitle), CardContent showing tool count Badge, status Badge (connected/disconnected)
- v1 is read-only — no edit capability (per deferred items). Future versions will allow adding/configuring servers.
- If servers array is empty (successful fetch but no servers): "No MCP servers configured. Add servers via claude.json or the MCP server package."

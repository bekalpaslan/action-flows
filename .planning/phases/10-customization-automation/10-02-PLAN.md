**Create packages/backend/src/services/skillService.ts:**

```typescript
import type { Storage } from '../storage/index.js';
import type { Skill, SkillId } from '@afw/shared';

export class SkillService {
  constructor(private storage: Storage) {}
```

Methods:
- `async createSkill(workbenchId: string, skill: Omit<Skill, 'id' | 'createdAt'>): Promise<Skill>` — generates UUID as SkillId, sets createdAt to ISO string, stores with key `skill:${workbenchId}:${skill.id}`, returns full Skill object
- `async listSkills(workbenchId: string): Promise<Skill[]>` — calls `this.storage.keys!(`skill:${workbenchId}:*`)`, fetches each with `this.storage.get!`, parses JSON, returns array sorted by createdAt descending
- `async getSkill(workbenchId: string, skillId: SkillId): Promise<Skill | null>` — single key lookup `skill:${workbenchId}:${skillId}`
- `async updateSkill(workbenchId: string, skillId: SkillId, updates: Partial<Pick<Skill, 'name' | 'description' | 'trigger' | 'action'>>): Promise<Skill | null>` — reads existing, merges, writes back, returns updated
- `async deleteSkill(workbenchId: string, skillId: SkillId): Promise<boolean>` — calls `this.storage.delete!`, returns result

Guard check: Every method validates that the storage key prefix matches the `workbenchId` parameter (D-06 scope isolation). The `updateSkill` and `deleteSkill` methods read the existing skill and verify `skill.workbenchId === workbenchId` before proceeding. If mismatch, throw Error(`Skill ${skillId} does not belong to workbench ${workbenchId}`).

Storage key pattern: `skill:{workbenchId}:{skillId}`

Use `crypto.randomUUID()` for ID generation (cast to SkillId).

**Create packages/app/src/stores/skillStore.ts:**

Use `zustand` with `persist` middleware (per RESEARCH.md Pattern 2). Use `Record<string, Skill[]>` instead of `Map` to avoid serialization issues (per RESEARCH.md Pitfall 6).

```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import type { Skill } from '@afw/shared';
import type { WorkbenchId } from '@/lib/types';
```

Interface:
```typescript
interface SkillState {
  skills: Record<string, Skill[]>;  // keyed by workbenchId
  loading: boolean;
  error: string | null;
  loadSkills: (workbenchId: WorkbenchId) => Promise<void>;
  addSkill: (workbenchId: WorkbenchId, skill: { name: string; description: string; trigger: string; action: string }) => Promise<void>;
  updateSkill: (workbenchId: WorkbenchId, skillId: string, updates: Partial<{ name: string; description: string; trigger: string; action: string }>) => Promise<void>;
  deleteSkill: (workbenchId: WorkbenchId, skillId: string) => Promise<void>;
  getSkillsByWorkbench: (workbenchId: WorkbenchId) => Skill[];
}
```

Implementation:
- `loadSkills`: `GET /api/skills/${workbenchId}` — on success, set `skills[workbenchId] = data.skills`
- `addSkill`: `POST /api/skills/${workbenchId}` with JSON body — on success, prepend new skill to `skills[workbenchId]`, optimistic update with revert on error
- `updateSkill`: `PUT /api/skills/${workbenchId}/${skillId}` — on success, replace skill in array
- `deleteSkill`: `DELETE /api/skills/${workbenchId}/${skillId}` — on success, filter out skill
- `getSkillsByWorkbench`: returns `skills[workbenchId] ?? []`

Persist config: `{ name: 'afw-skills' }` — use default JSON serializer since Record (not Map) is used.

Error handling: Set `error` string on fetch failures, clear on success. Set `loading: true` during loadSkills, `false` after.

Export: `export const useSkillStore = create<SkillState>()(persist(...))`

**Create packages/backend/src/services/forkMetadataService.ts:**

Per RESEARCH.md Pattern 5 (Fork Metadata as Separate Entity):

```typescript
import type { Storage } from '../storage/index.js';
import type { ForkMetadata, ForkId, MergeResolution } from '@afw/shared';
```

Class `ForkMetadataService`:
- Constructor: `private storage: Storage`
- Does NOT take sessionManager — fork creation is initiated by the route layer which calls sessionManager.forkSession separately. This service only manages metadata.

Methods:
- `async createFork(opts: { parentSessionId: string; forkSessionId: string; workbenchId: string; description: string; forkPointMessageId?: string }): Promise<ForkMetadata>`:
  - D-14: `description` is required — throw if empty
  - Generates UUID as ForkId
  - Creates ForkMetadata with status 'active', createdAt ISO
  - Stores at key `fork:${opts.parentSessionId}:${metadata.id}`
  - Returns metadata

- `async listForks(parentSessionId: string): Promise<ForkMetadata[]>`:
  - Keys pattern `fork:${parentSessionId}:*`
  - Parse, filter out status 'abandoned', return sorted by createdAt asc

- `async getFork(forkId: ForkId): Promise<ForkMetadata | null>`:
  - Scan all fork keys (keys `fork:*:${forkId}`) — since forkId is globally unique, check prefixed keys
  - Alternative: store a reverse index `forkIndex:${forkId}` -> storage key. Use this for O(1) lookup.
  - Return null if not found

- `async updateForkDescription(forkId: ForkId, description: string): Promise<ForkMetadata | null>`:
  - D-14: description cannot be empty
  - Read, update, write back

- `async mergeFork(forkId: ForkId, resolution: MergeResolution, manualContent?: string): Promise<ForkMetadata | null>`:
  - D-13: v1 implementation — simplest strategy per RESEARCH.md Open Question 3
  - Read fork metadata, validate status is 'active'
  - Resolution handling:
    - `'theirs'`: Replace parent's session state with fork's session content (conceptually: mark fork as merged, parent session now points to fork session ID for continued use)
    - `'parent'`: Keep parent unchanged, just mark fork as merged (discard fork changes)
    - `'manual'`: Store manualContent as a note on the merge (v1: same as 'parent' but with annotation)
  - Set fork.status = 'merged', fork.resolvedAt = now
  - Write back
  - Return updated metadata

- `async discardFork(forkId: ForkId): Promise<boolean>`:
  - Read fork, set status = 'abandoned', write back
  - Returns true if found and updated

Express Router factory: `export default function createForksRouter(forkMetadataService: ForkMetadataService, sessionManager: any): Router`

Routes:

- `POST /:forkId/merge` — body: `{ resolution: MergeResolution, manualContent?: string }`. Returns `{ success: true, fork }` or 404
- `DELETE /:forkId` — marks as abandoned. Returns `{ success: true }` or 404

**Create packages/app/src/stores/forkStore.ts:**

Zustand store (no persist — forks are ephemeral session data, re-fetched from backend):

```typescript
import { create } from 'zustand';
import type { ForkMetadata, MergeResolution } from '@afw/shared';
```

State interface (per UI-SPEC State Management):
- `forksByParent: Record<string, ForkMetadata[]>` (key: parentSessionId)
- `activeBranchBySession: Record<string, string>` (key: parentSessionId, value: active sessionId — either parentSessionId for "Main" or a fork's forkSessionId)
- `loading: boolean`, `error: string | null`
- `forkUnavailable: boolean` — set to true when backend returns 503 FORK_SESSION_UNAVAILABLE. Used by ForkDialog to show degradation message.
- `loadForks(parentSessionId: string): Promise<void>` — GET /api/forks/${parentSessionId}
- `createFork(parentSessionId: string, workbenchId: string, description: string, forkPointMessageId?: string): Promise<ForkMetadata | null>` — POST /api/forks. On success: append to forksByParent[parentSessionId], switch active branch to new fork, return metadata. On 400 (empty description), set error. **On 503 (FORK_SESSION_UNAVAILABLE)**: set `forkUnavailable = true`, set error to the message from the response. Return null.
- `switchBranch(parentSessionId: string, targetSessionId: string): void` — sets activeBranchBySession[parentSessionId] = targetSessionId
- `mergeFork(forkId: string, resolution: MergeResolution, manualContent?: string): Promise<void>` — POST /api/forks/${forkId}/merge, on success: update fork status in local state, switch back to main
- `discardFork(forkId: string): Promise<void>` — DELETE /api/forks/${forkId}, remove from local state, switch to main if was active
- `renameFork(forkId: string, description: string): Promise<void>` — PUT /api/forks/${forkId}/description
- `getForksForSession(parentSessionId: string): ForkMetadata[]` — returns forksByParent[parentSessionId] ?? []
- `getActiveBranch(parentSessionId: string): string` — returns activeBranchBySession[parentSessionId] ?? parentSessionId
- `isForkPoint(parentSessionId: string, messageId: string): boolean` — checks if any fork's forkPointMessageId matches

Export: `export const useForkStore`

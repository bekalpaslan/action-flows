    **useChatMessages.ts** -- WebSocket subscription mapping session:message events to chatStore (per RESEARCH Pattern 1).

    Follow the useSessionEvents.ts pattern: useEffect with wsClient.subscribe on '_system' channel, cleanup on unmount.

    ```typescript
    import { useEffect } from 'react';
    import { wsClient } from '../lib/ws-client';
    import { useChatStore } from '../stores/chatStore';
    import { parseAskUserQuestion } from '../lib/chat-types';
    import type { WSEnvelope } from '@afw/shared';
    import type { WorkbenchId } from '../lib/types';
    ```

    The hook subscribes to `_system` channel and filters for `session:message` type envelopes. The payload shape is `{ workbenchId: string, message: SDKMessage }` where SDKMessage is the raw Agent SDK message.

    **WorkbenchId normalization (Issue #4):** The WS payload `workbenchId` comes as a plain `string` from the backend, not a typed `WorkbenchId`. Cast via `String(payload.workbenchId) as WorkbenchId` after verifying it is a valid workbench ID string. This prevents silent drop if the backend serializes workbenchId differently. Add a guard: `if (!['work','explore','review','pm','settings','archive','studio'].includes(wbId)) return;`

    Handle these SDK message types (per RESEARCH Pattern 1):

    1. **`message.type === 'assistant'`**: Extract `message.message.content` blocks array.
       - For each block where `block.type === 'text'`: call `store.addMessage(workbenchId, { id: crypto.randomUUID(), role: 'agent', content: block.text, timestamp: new Date().toISOString(), status: 'complete' })`
       - For each block where `block.type === 'tool_use'`:
         - If `block.name === 'AskUserQuestion'`: Parse via `parseAskUserQuestion(block.input)`. If result is non-empty, call `store.addMessage` with `askUserQuestion: { toolCallId: block.id, question: parsedQuestions[0], response: null, submitted: false }`
         - Otherwise: call `store.updateMessage` to add to `toolCalls` array: `{ id: block.id, name: block.name, input: JSON.stringify(block.input), output: null, status: 'running' }`

    2. **`message.type === 'stream_event'`**: Handle content_block_start, content_block_delta, content_block_stop.
       - `content_block_start` with `content_block.type === 'text'`: `store.addMessage(workbenchId, { id: messageUuid-eventIndex, role: 'agent', content: '', status: 'streaming' })` and `store.setStreaming(workbenchId, true)`
       - `content_block_delta` with `delta.type === 'text_delta'`: `store.appendStreamChunk(workbenchId, messageId, delta.text)`
       - `content_block_stop`: `store.updateMessage(workbenchId, messageId, { status: 'complete' })` and `store.setStreaming(workbenchId, false)`

    3. **`message.type === 'result'`**: `store.setStreaming(workbenchId, false)`

    Use `useChatStore.getState()` (not selector) to avoid re-subscription on store changes, same pattern as useSessionEvents. Generate message IDs with `crypto.randomUUID()`. Use a `currentStreamingId` ref to track the active streaming message ID for delta/stop matching.

    Guard all message parsing with try/catch -- never crash on unexpected message format. Log unknown message types at debug level.

    **Extract the message handler function** as a standalone exported function (e.g., `handleSessionMessage`) so it can be unit-tested without rendering a React component. The hook's useEffect calls this function.

    **useAutoScroll.ts** -- per RESEARCH Pattern 4.

    ```typescript
    import { useRef, useCallback } from 'react';
    import type { RefObject } from 'react';
    ```

    Takes `messageListRef: RefObject<HTMLDivElement | null>` as parameter. Returns `{ handleScroll, scrollToBottom, onNewMessage, isUserScrolled: RefObject<boolean>, unreadCount: RefObject<number> }`.

    - `handleScroll`: Calculate `distanceFromBottom = el.scrollHeight - el.scrollTop - el.clientHeight`. If > 100: `isUserScrolled.current = true`. If <= 100: `isUserScrolled.current = false; unreadCount.current = 0`.
    - `scrollToBottom`: `messageListRef.current?.scrollTo({ top: scrollHeight, behavior: 'smooth' })`. Reset isUserScrolled and unreadCount to 0.
    - `onNewMessage(isUserMessage: boolean)`: If isUserMessage or not isUserScrolled: scrollToBottom. Otherwise: increment unreadCount.

    **useChatSend.ts** -- Send messages via WebSocket.

    ```typescript
    import { useCallback } from 'react';
    import { wsClient } from '../lib/ws-client';
    import { useChatStore } from '../stores/chatStore';
    import type { WorkbenchId } from '../lib/types';
    ```

    Returns `{ sendMessage: (workbenchId: WorkbenchId, text: string) => void, sendAskUserResponse: (workbenchId: WorkbenchId, messageId: string, toolCallId: string, response: unknown) => void }`.

    `sendMessage`:
    1. Add user message to chatStore: `store.addMessage(workbenchId, { id: crypto.randomUUID(), role: 'user', content: text, timestamp: new Date().toISOString(), status: 'sent' })`
    2. Clear input: `store.setInputValue(workbenchId, '')`
    3. Send via WebSocket: `wsClient.send({ type: 'chat:send', payload: { workbenchId, text } })`

    `sendAskUserResponse`:
    1. Mark as submitted in store: `store.submitAskUserResponse(workbenchId, messageId, JSON.stringify(response))`
    2. Send via WebSocket: `wsClient.send({ type: 'chat:ask-user-response', payload: { workbenchId, toolCallId, response } })`

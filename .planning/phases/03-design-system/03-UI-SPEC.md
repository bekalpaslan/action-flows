# Phase 3 -- UI Design Contract

> Visual and interaction contract for the design system phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase does NOT build pages or layouts. It builds the token layer and component library that all subsequent phases compose from. The contract defines the exact values migrating from design-tokens.css into Tailwind v4 @theme and the visual contracts for all 12 core components.

## Typography

| Role | Size | Weight | Line Height | Letter Spacing | Usage |
|------|------|--------|-------------|----------------|-------|
| Label/Caption | 13px | 400 (normal) | 1.5 | normal | Secondary labels, captions, badges, fine print, tab triggers |
| Body | 15px | 400 (normal) | 1.5 | normal | Default body text, paragraphs, inputs, descriptions |
| Heading | 20px | 600 (semibold) | 1.2 | -0.025em | Section headings, card titles, page headings |
| Display | 28px | 600 (semibold) | 1.2 | -0.05em | Hero/display text, major page titles |

**Font families:**
- `--font-sans`: -apple-system, BlinkMacSystemFont, "SF Pro Display", "Segoe UI", "Roboto", "Helvetica Neue", Arial, sans-serif
- `--font-mono`: "SF Mono", "Monaco", "Consolas", "Liberation Mono", "Courier New", monospace

**Constraints for component library:** Components use exactly 2 weights: 400 (normal) for body text and 600 (semibold) for headings/display. No other weights in the component library.

**Migration from 8-size scale:** The original design-tokens.css defined 8 sizes (11-34px). This contract collapses them to 4 canonical sizes. Components that previously used removed sizes map as follows: 11px -> 13px (Label/Caption), 17px -> 15px (Body), 24px -> 20px (Heading), 34px -> 28px (Display). Weight 700 (bold) is removed; Display now uses 600 (semibold) for consistency.

## Component Manifest Contract

Per DESIGN-05 and D-08, a machine-readable manifest is required.

**Format:** TypeScript type exports + generated JSON file.

**Location:** `packages/app/src/components/ui/manifest.ts` (source of truth) and `packages/app/src/components/ui/manifest.json` (generated for agent consumption).

**Manifest entry schema per component:**

```typescript
interface ComponentManifestEntry {
  name: string;              // PascalCase component name
  importPath: string;        // "@/components/ui/{name}"
  description: string;       // One-line purpose
  variants: Record<string, string[]>;  // variant name -> allowed values
  defaultVariants: Record<string, string>;
  sizes: string[];           // Allowed size values
  subComponents: string[];   // Child components (e.g., CardHeader, CardContent)
  radixPrimitive: string | null;  // Package name or null
  props: Record<string, {    // Key props only (not every HTML attribute)
    type: string;
    required: boolean;
    description: string;
  }>;
}
```

## Tailwind v4 @theme Migration Contract

The existing 993-line design-tokens.css is deleted after migration. All values move into a Tailwind v4 CSS file using `@theme` directives. Theme switching uses `@variant` or CSS selector overrides for `[data-theme="light"]`.

### Required @theme sections:

1. **Colors** -- All palette values, semantic mappings for both themes
2. **Spacing** -- Full 4px-base scale (0 through 128px)
3. **Typography** -- Font families, sizes, weights, line heights, letter spacing
4. **Border radius** -- xs through full
5. **Shadows** -- Elevation scale + glow system
6. **Z-index** -- 10-level scale
7. **Transitions** -- Duration scale + easing functions

### Theme structure:

```css
@import "tailwindcss";

@theme {
  /* Palette, spacing, typography -- theme-independent values */
}

/* Dark theme (default) */
:root {
  /* Semantic color assignments for dark */
}

/* Light theme */
[data-theme="light"] {
  /* Semantic color assignments for light */
}

@media (prefers-reduced-motion: reduce) {
  :root {
    /* Zero all durations */
  }
}
```

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Focus visible | 2px solid accent outline, 2px offset, glow on dark backgrounds |
| Reduced motion | All animation durations to 0ms via `prefers-reduced-motion: reduce` |
| Touch targets | Minimum 44px hit area (WCAG 2.5.8) for all interactive elements |
| Color contrast | Text-primary on surface: minimum 4.5:1 ratio (WCAG AA) |
| ARIA | All Radix primitives provide correct ARIA roles, states, properties |
| Keyboard | All interactive components fully keyboard-navigable (Tab, Enter, Space, Escape, Arrow keys where applicable) |
| Screen reader | All components announce state changes (checked, expanded, selected) |
| Icon-only buttons | Must include `aria-label` or visually-hidden `<span className="sr-only">` child. No icon-only button without a text alternative. |
